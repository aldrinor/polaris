# Phase 4 Sol verdict — execution charter and master action plan

## 1. Decision

The build should be organized around one typed, pre-generation `AnalyticalContract`, not fourteen switches and not fourteen prompt fragments. The contract is created from the question and admitted evidence, refined by retrieval and relation analysis, projected into the outline and paragraph/claim plans, consumed by the active verified producer, and audited semantically after generation. This is the only design consistent with both investigators' shared verdict and with the measured failure of the independent-lever approach (`PIPELINE_GAP_AUDIT.md:24-28,50-55,668-679,810-820`).

The first production wave should **not** be U1 or U3 alone. U1 alone would be another advisory writer rule that can miss `_compose_section_per_basket`; U3 alone would produce relations without trustworthy admission, semantic obligation ownership, or a block carrier. The first wave must establish the shared contract, admission/coverage semantics, marginal ownership, and structure carrier. U1/U2/U3 then become the immediate second wave and are the first direct attack on the two 0.0800 Insight cells (`PIPELINE_GAP_AUDIT.md:57-72,86-170,174-219,742-759,810-820`).

This is design only. No faithfulness file, pipeline file, benchmark file, or configuration file is modified by this verdict.

# DELIVERABLE A — EXECUTION CHARTER

## A1. Generalization gate

### A1.1 Admission rule

A fix is general only if all of the following are true before it may enter a build wave:

1. Its runtime content is derived from the current question, extracted constraints, admitted evidence, and evidence relations. Runtime code and prompts contain no task ID, benchmark-specific answer content, labor-market/AI/4IR literal, venue whitelist, sector list, or other domain literal. The only fixed vocabularies permitted are research **roles**, query **types**, relation **types**, epistemic **statuses**, and contract **states**. This follows the audit's explicit rule that content be prompt/evidence-derived while only role/type vocabularies are fixed (`PIPELINE_GAP_AUDIT.md:302-316,376-389,413-416,516-532,810-820`).
2. It contains no runtime count whose purpose is to force a score, breadth, paragraph, source, section, table, or citation target. Numeric values mandated by this initiative—two held-out corpora, at least 3v3 judge draws, and the ±0.027 decision band—are test protocol values, not generation behavior. Any other test threshold must be a named acceptance constant in test fixtures with its evidence/rationale beside it; it must not enter runtime selection or prose generation (`SCORING_SPEC.md:236-245`; `PIPELINE_GAP_AUDIT.md:802-806`).
3. It introduces no new independently toggled behavior flag. In particular, no new `PG_*` flag with a quality adjective such as “max,” “full,” “rich,” “smart,” “deep,” or “enhanced” is allowed. The behavior is a required projection of the one contract when its semantic preconditions are present. This prevents recurrence of the measured flat-to-negative flag slate (`PIPELINE_GAP_AUDIT.md:50-55,63-68,668-679`).
4. Its deterministic small test passes on at least two unrelated held-out questions/corpora chosen and frozen before implementation. Candidate benchmark tasks are:
   - task 100, a high-Insight conceptual analysis, as a positive analytical-role case;
   - task 91, a high-Comprehensiveness entity inventory, as a negative control for causal/mechanism-only roles;
   - task 73, an audience/readability-heavy practical task;
   - task 4, a scenario-and-mind-map task with a different language and output form.
   These tasks are documented as structurally and dimensionally different (`SCORING_SPEC.md:74-77`; `COMPETITOR_TEARDOWN.md:86-102`).
5. Role-triggered fixes pass both a positive and a negative activation test. For example, a mechanism spine must activate on a causal/impact question and remain absent on a pure inventory unless the evidence/question independently licenses one. The Phase-3 small tests explicitly require causal, descriptive, and missing-link fixtures (`PIPELINE_GAP_AUDIT.md:151-170`).
6. Every contract field has evidence ownership or an explicit non-claim state (`retrieval_needed`, `unsupported_disclosed`, or `not_applicable`). Empty strings, section presence, or prompt-word matches cannot count as semantic fulfillment (`PIPELINE_GAP_AUDIT.md:274-323`).

### A1.2 Reviewer audit

The reviewer must perform all of these checks:

- Grep the runtime diff, prompts, configs, and schema defaults for task IDs, benchmark names, literals copied from the held-out questions, domain terms from task 72, fixed breadth/source/paragraph targets, and newly added adjective flags. Test fixtures are reviewed separately and may contain their declared fixture content.
- Trace every fixed enum and confirm it is a role/type/status vocabulary, not answer content.
- Run the exact same small test on the two prespecified held-out corpora. A fix that passes task 72 but fails either held-out corpus is rejected as overfit.
- Run the negative-control question and verify that inapplicable roles remain absent rather than appearing as generic report boilerplate.
- Inspect a serialized contract and prove that every content-bearing field points to question spans, constraint IDs, evidence IDs/spans, or upstream contract IDs.
- Compare the emitted prose with the contract semantically: a nonempty section is never evidence that an obligation was fulfilled. The emitted proposition must entail the obligation-and-role pair (`coverage_obligations.audit_fulfillment` is currently toothless for exactly this reason; `PIPELINE_GAP_AUDIT.md:277-285,302-323`).

Failure of any item is an immediate no-go. There is no RACE probe for a fix that fails this gate.

## A2. Faithfulness firewall

### A2.1 Untouchable files and behavior

The following are outside the modification set for every wave:

- `src/polaris_graph/generator/provenance_generator.py`, including provenance rewriting, citation resolution, and sentence provenance verification.
- `src/polaris_graph/generator/clinical_generator/strict_verify.py`, including identifier, span, numeric, overlap, and entailment checks.

Phase 3 identifies these as claim-admission controls rather than score planners and explicitly says the fix belongs upstream (`PIPELINE_GAP_AUDIT.md:69-72`; see also `phase3_sol_verdict.md:15-19`). No wave may weaken, bypass, special-case, lower a threshold in, or reinterpret either component.

Every new factual or analytical sentence must still carry valid evidence markers. If strict verification removes a planned sentence, the response is to repair evidence ownership, regenerate from the plan, or disclose the gap; the sentence may not be restored after verification and the verifier may not be relaxed (`PIPELINE_GAP_AUDIT.md:724-729`).

### A2.2 Licensed-inference admission

U1's licensed inference is admitted only if a deterministic canary proves:

1. The inference's evidence-marker set is the union or a subset of the premise sentences' marker sets.
2. After normalization, every factual token in the inference already occurs in its cited premise sentences or their exact cited spans. Factual tokens include numerals, dates, units, named entities, identifiers, and content-bearing terms. Only a fixed connective/epistemic role vocabulary may be new.
3. No new URL, citation identity, population, measure, direction, magnitude, named entity, or boundary condition appears.
4. The inference is represented in the contract as synthesis-derived, with premise claim IDs and its permitted reasoning operator.

This implements the audit's “zero new factual token/number/entity” canary and union-marker rule without touching strict verification (`PIPELINE_GAP_AUDIT.md:111-127`).

If the canary fails, the inference is never sent to the producer. It is not rewritten after generation.

### A2.3 Absolute post-generation rule

No post-generation content edit is permitted. After the producer emits text, allowed render normalization is limited to layout:

- inserting or normalizing whitespace/newlines around already-emitted headings, paragraphs, and tables;
- preserving preplanned block boundaries and heading nodes;
- formatting an already-planned table without changing cell text.

The normalizer must prove that the ordered non-whitespace token stream is byte-for-byte identical before and after normalization. It may not add, remove, paraphrase, reorder, merge, or split sentence content. This is the layout-only exception described for U12 (`PIPELINE_GAP_AUDIT.md:572-592`). The existing post-assembly summary-table path is forbidden because it injects content after composition (`PIPELINE_GAP_AUDIT.md:223-270,668-675`).

## A3. Measurement gate

### A3.1 Tier 1 — deterministic proof first

Every lever in a wave must first pass its deterministic behavioral assertion. This proof must exercise the real seam named in the wave, including `_compose_section_per_basket` where prose is involved. A green unit test over an unused prompt or `_call_section` alone is not sufficient (`PIPELINE_GAP_AUDIT.md:57-62,802-806,810-820`).

Deterministic assertions are evaluated independently per U-item. One failed U-item blocks the whole wave; it may be removed and the wave redesigned, but it may not be carried into the RACE probe as an unproven passenger.

### A3.2 Tier 2 — assembled paired RACE probe

After all deterministic checks in a wave pass:

1. Build the **assembled wave**, not isolated flag combinations.
2. Compare it to the immutable `mf_baseline` champion: all initiative levers off, `PG_RENDER_BLOCKS` on, mean Overall approximately 0.5009. The champion's three draws and spread are the required control, not `champ_ourcorpus`, `faithoff_t72`, a degraded max arm, or a previous losing lever slate (`PIPELINE_GAP_AUDIT.md:39-53`).
3. Use the same frozen corpus snapshot, question, model/judge, reference, cleaner policy, and otherwise-identical configuration. Pair candidate and champion calls in a prespecified interleaved order so provider drift is shared.
4. Run at least three candidate draws and three champion draws. Freeze and hash the cleaned candidate/champion articles used for the scoring draws so cleaner randomness is not misattributed to the score judge; separately run one end-to-end cleaner robustness check.
5. Compute each pair's candidate-minus-champion delta for Overall and all four dimensions. Decide on the paired mean, never on a single draw.
6. A targeted dimension must improve by more than +0.027. Any dimension at or below −0.027 is a material regression and is an immediate no-go. A negative mean inside the band is inconclusive, not a pass: run only the prespecified confirmatory pairs, and do not promote unless the mean is nonnegative. This uses the measured ±0.027 noise bar rather than pretending a small fluctuation is signal (`SCORING_SPEC.md:236-245`; `PIPELINE_GAP_AUDIT.md:50-53,802-806`).
7. The cumulative candidate must also be compared with the last approved wave. This secondary comparison prevents a later cumulative wave from preserving a win over the old champion while erasing a previously banked dimension gain.

### A3.3 Stop rule

A lever or wave does not ship when:

- any deterministic structural/semantic/faithfulness assertion fails;
- a targeted dimension does not clear +0.027 after the prespecified paired probe;
- any RACE dimension materially regresses by −0.027 or worse;
- the FACT precision/non-inferiority gate for a citation-affecting wave fails;
- the generalization or firewall audit fails.

There is no “ship because Overall rose” exception for a dimension regression. There is no build-all-and-guess: build-all-then-measure means deterministic tests attribute each lever's behavior, and the paired RACE probe evaluates the integrated wave (`PIPELINE_GAP_AUDIT.md:802-806`).

## A4. Shared-contract rule

### A4.1 One object

Every wave extends or consumes one versioned `AnalyticalContract`. Proposed top-level fields are:

| field | required content and invariant |
|---|---|
| `question` | question spans, semantic operators, requested deliverable, named concepts/classes, constraints |
| `admission` | each source/evidence row's eligibility decision, evidence for that decision, and reader-safe disclosure |
| `obligations` | question-derived dimension/role obligations with `supported`, `retrieval_needed`, `unsupported_disclosed`, or `not_applicable` |
| `mechanisms` | evidence-backed chain links, moderators, net-direction condition, missing-link boundaries |
| `relations` | convergence, qualified divergence, or non-comparability edges; direct conflict is represented as a qualified-divergence subtype with both poles |
| `contexts` | evidence-derived context/regime attributes and shared comparison schemas |
| `measurements` | construct, unit, basis, margin, population, period, affected party, and observation/model/forecast status |
| `inferences` | premise claim IDs, reasoning operator, boundary conditions, epistemic status, falsifier/resolving observation |
| `implications` | upstream mechanism/relation IDs, affected context, direction, trade-off, evidence grade, resolving observable |
| `sections` | role, owned obligations/relations, evidence/basket IDs, ordered paragraph-block IDs |
| `blocks` | reader question, analytical move, transition relation, claim IDs, optional planned table ID |
| `claims` | atomic proposition frame, exact evidence span ownership, marker set, factual-token inventory, canonical URL/dedup identity where applicable |
| `audit` | semantic fulfillment, generalization, canary, and render-preservation results |

This unifies the Phase-3 required components—mechanism chains, coverage ledger, relation graph, inference labels, admission ledger, paragraph blocks, and active-producer consumption (`PIPELINE_GAP_AUDIT.md:810-820`).

### A4.2 Producer rule

The contract is projected per section and per block directly into `src/polaris_graph/generator/verified_compose.py:_compose_section_per_basket` (currently defined at line 2841 and imported by `multi_section_generator.py:60-69`). `multi_section_generator.py:_run_section` selects that primary branch at `:6477-6713`; `_call_section` is the fallback at `:6714-6728`. A feature that reaches only `_call_section`, sidecar prompt text, telemetry, or an unused outline field is unwired and fails the gate (`PIPELINE_GAP_AUDIT.md:57-62`).

The producer must return block/claim IDs alongside its draft so semantic acceptance can map surviving verified sentences back to obligations. The contract does not bypass provenance or strict verification.

### A4.3 No independent-flag path

Existing useful ideas are absorbed into contract builders/validators:

- contradiction mining becomes relation-edge construction;
- relation packs become contract relation/context projections;
- coverage obligations become semantic contract obligations;
- scope deepening becomes a required transition for `retrieval_needed`;
- section structure becomes contract block preservation;
- narrative attribution becomes a claim/source projection.

The sentence-span synthesis-table constructors are retired from the judged body, not re-flagged. Their diagnostics may remain in a sidecar that never reaches RACE (`PIPELINE_GAP_AUDIT.md:668-679`).

## A5. No-regression/champion-protection gate

The champion is the exact `mf_baseline` artifact/config family: mean Overall 0.5009, all initiative levers off, `PG_RENDER_BLOCKS` on (`PIPELINE_GAP_AUDIT.md:41-53,63-68`). Before build:

1. Archive hashes of the corpus snapshot, raw reports, cleaned reports, reference, judge/model settings, resolved config, and score outputs for all champion draws.
2. Preserve the champion path as a reproducible comparison target. Do not overwrite its artifacts or silently redefine its config.
3. Measure every cumulative wave directly against that champion, even if a previous wave already won.
4. Also compare against the prior approved wave to prevent loss of banked gains.
5. Promote atomically only after both comparisons and all deterministic gates pass. A failed candidate leaves the champion selected.
6. Production admission still requires the firewall even though the historical champion telemetry had strict verification disabled; the champion is the RACE comparison oracle, not authority to weaken faithfulness (`PIPELINE_GAP_AUDIT.md:63-72`).

### Why weight alone does not put U1/U3 first

- **U1 alone is not the smallest safe win.** Without first-class blocks and claim ownership, its “paragraph closing” behavior is not reliably represented in `_compose_section_per_basket`, and a prompt-only implementation repeats the failure mode.
- **U3 alone is not a shippable report behavior.** A nonempty relation ledger is a deterministic success, but it earns no score until admitted evidence, semantic ownership, and the active producer turn it into verified explanatory prose.
- **Wave 1 is the smallest high-confidence first shipment.** It replaces the failed architectural assumptions—section presence as coverage, ungoverned source admission, route-all prose, and advisory structure—with a single auditable carrier. - **Wave 2 follows immediately** and is the first direct Insight ceiling play. No discretionary polish or unrelated work belongs between Waves 1 and 2.

## B7. Operator decisions required before build

1. **Held-out generalization set.** Approve the exact two-or-more frozen benchmark tasks/corpora per wave. Recommended core set: task 100 as analytical positive, task 91 as inventory negative control, with task 73 and task 4 added for readability/output-form/language coverage.
2. **Eligibility shrink tolerance and retrieval wall.** Decide how much corpus shrink from exclusive source constraints is acceptable before the system must keep deepening versus disclose an unfillable obligation. The plan will not admit wrong-type evidence merely to preserve breadth.
3. **Legacy lever retirement policy.** Approve:
   - rewire contradiction mining/relation packs/coverage obligations into the shared contract;
   - retire `PG_SYNTHESIS_TABLE_CONSTRUCT` and `PG_SUMMARY_TABLE_COMPOSE` from the judged body, optionally retaining diagnostic sidecars;
   - replace independent structure/deepening/eligibility switches with contract-required behavior when semantic preconditions hold.
4. **Visible discourse labels.** Choose whether planned deduction/epistemic blocks visibly use labels such as `Implication`, `Evidence status`, and `What would resolve this`, or whether the writer renders natural prose while preserving the same typed contract. The safer audit surface is visible labels; the readability trade-off must be measured.
5. **Inconclusive paired-result policy.** Approve the prespecified maximum confirmatory pair count when a targeted delta or small negative remains inside ±0.027. The recommendation is a fixed maximum set before looking at results; no open-ended redraw-until-win.
6. **FACT canonical-URL policy.** Choose preference order among DOI resolver, journal landing page, and stable full-text repository when multiple URLs represent the same work, and the maximum allowed age of the reachability preflight.
7. **FACT baseline identity.** The RACE champion is clear, but FACT artifacts show different precision/volume points (faithoff 11 at 100%, b0fact 16 at 94.1%, b1fact 31 at 68.9%). Select the production FACT non-inferiority baseline for Wave 5 (`PIPELINE_GAP_AUDIT.md:626-637`).
8. **Promotion strictness across non-target dimensions.** This charter recommends no material regression (≤−0.027) and no promotion on an unresolved negative mean. Confirm that conservative policy rather than permitting an Overall-for-dimension trade.
