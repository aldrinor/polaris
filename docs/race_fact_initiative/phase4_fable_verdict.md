# DELIVERABLE A — EXECUTION_CHARTER

Five gating rules. Every fix, in every wave, must pass ALL FIVE before it ships. Each rule states (i) the
enforceable rule, (ii) how a reviewer AUDITS it mechanically, (iii) the stop condition.

## R1 — GENERALIZATION GATE (no task-72 overfit ships)

**Rule.** A fix is admissible only if ALL of the following hold:

1. **Zero task/domain literals** in code, prompts, and config it introduces. Banned classes: task-72 domain nouns
   (e.g. labor/industry/4IR/journal-name strings), benchmark-task identifiers, venue whitelists, and any
   domain-vocabulary lookup table. The only FIXED vocabularies permitted are the audited **role/type/status
   vocabularies**, which are properties of research questions and evidence, not domains
   (`PIPELINE_GAP_AUDIT.md:306-316`): analytical roles {mechanism, cross-context comparison,
   consensus/disagreement, implications}; query types {factual, causal, comparative, critical}; comparability
   boundaries {population, method, level, period, measure} (the same fields
   `relation_evidence_packs._attributes` already aliases, `PIPELINE_GAP_AUDIT.md:210-212`); the three-level
   epistemic-status vocabulary (`:381-389`). CONTENT always comes from the question + admitted evidence at run
   time.
2. **Zero magic counts** (no hardcoded section counts, industry counts, word targets) and **zero adjective flag
   names** (no `PG_BETTER_X`-style levers; components are named by the seam/object they build).
3. **Held-out proof.** The fix's deterministic small test (the audit's SMALL TEST for that gap) passes on
   task 72 AND on **≥2 unrelated held-out benchmark tasks** with materially different weight profiles.
   Named candidates from the frozen criteria distribution (`SCORING_SPEC.md I.5:74-77`):
   - **task 91** (Saint Seiya inventory; weights .37 Comp / **.11 Insight** / .32 Inst / .20 Read) — the
     anti-analytical extreme; conditional analytical triggers (U2 mechanism role, U5 analytical-role ledger rows)
     must NOT fire or must degrade gracefully;
   - **task 100** (AI & interpersonal relationships; .29/.40/.16/.15) — the high-Insight analytical extreme;
   - **task 73** (novice EFL teachers; Read **.25**, highest in corpus) — audience/readability stressor;
   - **task 51** (Japan elderly consumption; data-heavy, heading-sparse reference) — data-presentation stressor;
   - **task 4** (zh, gold + mind-map deliverable; .20/.38/.26/.16) — language + explicit format-instruction
     stressor.
   Mandatory pair: **91 + 100** (opposite Insight extremes). Recommended add: 73 or 51. Operator picks the final
   set (Open Question OQ-1).

**Audit procedure.** (a) `grep` the diff for the banned classes: any token that appears in the task-72 prompt or
criteria but not in the fixed role/type/status vocabularies fails review; reviewer greps for digits in prompt/
config additions to catch magic counts. (b) Run the fix's deterministic test on the held-out tasks' frozen
corpora/outlines; the trigger-conditionality check on task 91 is mandatory for any analytically-conditioned
component. (c) Confirm the component reads its content from the constraint extractor
(`compose_agentic_report_s3gear329.py:398-410`) or admitted evidence rows, never from its own literals.

**Stop.** Any literal/magic-count/adjective-name finding, or a held-out deterministic-test failure, blocks the
wave until fixed. No exceptions for "temporary" literals.

## R2 — FAITHFULNESS FIREWALL (untouchable set + licensed inference + no post-gen edits)

**Rule.**
1. **Untouchable files.** No fix modifies `provenance_generator.py` (rewrite + citation resolution,
   `:3714-3943`, `:4637-5252`) or `clinical_generator/strict_verify.py` (identifier/span/numeric/overlap checks,
   `:387-574`). These are claim-admission controls, not score planners (`PIPELINE_GAP_AUDIT.md:69-73`). No new
   entailment/NLI machinery is introduced anywhere (standing operating rule; the champion runs strict-verify OFF
   and the initiative's fixes must be valid under both strict-verify configurations).
2. **Every sentence still carries evidence markers.** All new writer obligations produce sentences that carry
   `[ev_XXX]` markers exactly as today (`multi_section_generator.py:3517-3519`).
3. **Licensed-inference admission check (the U1 canary).** The ONE new sentence class — the paragraph-closing
   inference — is admitted only under the deterministic **zero-new-factual-token canary**: the closing sentence
   introduces no numeric token, no new proper-noun/entity, and no factual claim absent from the preceding cited
   sentences of that same paragraph, and carries the union of that paragraph's markers
   (`PIPELINE_GAP_AUDIT.md:111-128`). The same canary gates U7's labeled propositions (each grounded in ≥2 cited
   findings; the label explicitly marks the report's own inference as inference — the faithful way to be novel,
   `:381-389`) and U10's implication objects (derived only from upstream cited propositions, `:469-476`).
4. **NO post-generation content edit, EVER.** No sentence is added, removed, or reworded after composition.
   The NVIDIA-style rubric rewrite is the one competitor move structurally forbidden to us
   (`PIPELINE_GAP_AUDIT.md:727-729`); its surfaces are absorbed pre-generation into U1/U2/U4/U7/U10.
   **Sole labeled exception — layout-only render normalization**: deterministic markdown layout operations on
   writer-emitted text with ZERO word changes and ZERO content drops (split run-in heading lines at the
   heading/body boundary, enforce blank lines around headings), the same category as the shipping
   `_materialize_paragraph_breaks` (`multi_section_generator.py:3686-3692`) and `markdown_table_normalizer`
   (`PIPELINE_GAP_AUDIT.md:578-583`). Any operation that touches a word, a number, a marker, or sentence
   membership is a content edit and is banned. The normalizer must ship with a diff-auditor asserting
   token-multiset equality before/after (layout characters excluded).

**Audit procedure.** (a) `git diff --stat` of the wave must show zero lines changed in the untouchable files.
(b) The canary runs as a deterministic lint on every composed artifact (it is Wave 1's shipped test).
(c) The render normalizer's token-multiset equality check runs on every assembly.
(d) Reviewer greps new prompts for instructions that would license out-of-evidence content beyond the one
inference class.

**Stop.** Any untouchable-file diff, any canary failure, any normalizer token change → the wave does not ship.
If strict verification removes a planned sentence, the lawful response is stronger evidence ownership or
regeneration from the plan, never weakening faithfulness or editing the finished report
(`PIPELINE_GAP_AUDIT.md:724-726`).

## R3 — MEASUREMENT GATE (deterministic first, paired probe second, stop rule)

**Rule.** Two-tier proof, in order, per `PIPELINE_GAP_AUDIT.md:802-806`:
1. **Tier 1 — deterministic structural assertion FIRST.** Every component ships its audit-named SMALL TEST as a
   zero/low-LLM deterministic assertion (free, noise-immune). A component whose deterministic test fails does not
   proceed to any RACE spend. Concrete gates per wave in §B3.
2. **Tier 2 — paired same-judge multi-draw RACE probe.** ≥3 candidate draws vs ≥3 champion draws (**3v3
   minimum**), same corpus, same judge, otherwise-identical config, decided on the **paired MEAN per-dimension
   delta clearing ±0.027** (the measured baseline spread 0.4922–0.5088 IS this bar,
   `phase3_fable_verdict.md:40`). Never a single-draw comparison.
3. **Build-all-then-measure.** Deterministic tests gate each component individually; the RACE probe measures the
   assembled wave (and, at the end, the assembled ladder), per the standing build-all-then-measure order and
   `PIPELINE_GAP_AUDIT.md:805-806`. Within-wave components are not RACE-probed one at a time.
4. **STOP RULE.** A component whose deterministic test fails does not ship. A wave whose paired probe shows any
   dimension's paired mean regressing beyond −0.027 does not ship as-is: the wave is decomposed (components
   staged off via the contract's component switches, §R4), the offending component identified by its
   deterministic telemetry, and either repaired or retired. A wave that is flat (all paired means inside ±0.027,
   none negative beyond it) MAY ship only if its deterministic gates all pass AND it is a declared dependency of
   a later wave (foundation shipping), with the flat result recorded — never spun as a win.

**Audit procedure.** Probe manifests record judge model, corpus hash, config hash, draw IDs; reviewer recomputes
the paired means from `results/race/*/raw_results.jsonl`. Any probe missing its paired champion arm is invalid.

**Stop.** As in rule 4 above; additionally, any probe run against a non-champion comparator is void (see R5).

## R4 — THE SHARED-CONTRACT RULE (one contract, consumed by the active producer, audited semantically)

**Rule.** Both Phase-3 investigators converged independently (`PIPELINE_GAP_AUDIT.md:24-28`, `:810-820`): the fix
is **ONE pre-generation analytical contract** — hereafter the **Analytical Contract (AC)** — not a set of
independent flags. Therefore:
1. **One object.** The AC is a single per-run artifact holding the components the waves build: coverage-dimension
   ledger (U5), mechanism/causal-chain spine (U2), divergence ledger + relation graph (U3), comparison groups /
   table plans (U4/U6), epistemically-labeled inference plans (U7), concept-role spine (U8), measurement/effect
   ontology (U9), implication objects (U10), source-admission ledger (U11), block/layout plan (U12), routing
   ownership (U13), external-citation claim contract (U14).
2. **One consumption point.** Every component is consumed by the **active producer,
   `_compose_section_per_basket`** — the verified-compose primary branch — not only by `_call_section`, which is
   the `else` branch (`multi_section_generator.py:6630-6728`; `PIPELINE_GAP_AUDIT.md:58-61`). A prompt addendum
   routed only to `_call_section` misses the producer that emits the sentences; this exact miswiring is a proven
   cause of the measured-flat lever set (`:668-679`). Definition of done for every wave includes a wiring
   assertion that the component's payload is observable in the active producer's inputs.
3. **Semantic audit.** Fulfillment of any AC obligation is SEMANTIC: the planned/emitted proposition must ENTAIL
   the obligation (dimension-role pair, channel reference, boundary term, label, policy clause), never "the bound
   section is non-empty" — the toothless-audit failure of `coverage_obligations.py:139-158`
   (`PIPELINE_GAP_AUDIT.md:277-283`). ("Semantic" here means checked at proposition level by the audit's
   deterministic tests and fixtures — NOT an NLI/entailment model; R2 bans those.)
4. **No independent flags.** New behavior enters as an AC component with a component switch used ONLY for staged
   measurement (R3's decomposition) and rollback; the switch namespace is the AC's, not a new `PG_*` adjective
   flag. The independent-flag path is what measured flat (`:24-28`, `:818`).
5. **Advisory prose is not a fix.** A wave that only adds writer-prompt sentences without changing evidence
   ownership or plan objects must justify why it will not repeat the measured-flat failure — U1 qualifies because
   it changes the writer's licensed sentence CLASS and ships a deterministic behavioral gate, not because it is
   prose.

**Audit procedure.** (a) Wiring assertion: dump the active producer's assembled inputs on a fixture run and
assert the component payload is present (Sol's U2 test pattern, `PIPELINE_GAP_AUDIT.md:164-170`). (b) Grep for
new top-level env flags outside the AC namespace. (c) Each obligation's fulfillment test is proposition-level.

**Stop.** A component not consumed by `_compose_section_per_basket`, or audited only by section-nonemptiness,
does not ship.

## R5 — NO-REGRESSION GATE (the champion is the only comparator, and it is protected)

**Rule.**
1. **The comparator is mf_baseline** — all levers OFF except `PG_RENDER_BLOCKS=1`, strict-verify bypassed as
   measured — at Overall mean **0.5009** (draws 0.5088/0.5017/0.4922; `PIPELINE_GAP_AUDIT.md:46`, `:63-68`;
   `phase3_fable_verdict.md:42`). Every wave's Tier-2 probe runs candidate-vs-champion, paired, same judge, same
   corpus. Measuring against `mf_max`, `7phase`, `faithoff_t72`, or any other strawman is void — the champion is
   the HIGHEST measured config; beating a weaker arm proves nothing.
2. **Ship criterion.** A wave ships only if (a) paired mean Overall ≥ champion within noise (no Overall
   regression beyond −0.027), (b) no dimension regresses beyond −0.027 paired mean, and (c) its claimed win —
   if a win is claimed — clears +0.027 on the targeted dimension(s).
3. **Champion protection.** (i) The champion config (`resolved_lever_states` snapshot,
   `outputs/race_max_focus/mf_baseline-20260723T152731Z/draw_3/compose_summary.json`) is pinned in VCS as a named
   frozen config; (ii) its three baseline draws and their scores are retained as the reference distribution;
   (iii) the champion path stays runnable at all times — AC components default OFF until their wave ships, so
   the shipped pipeline with an empty AC must reproduce the champion behavior byte-for-byte at the config level;
   (iv) after each shipped wave, the new assembled config becomes the NEW champion for the next wave's probes
   (ladder discipline), with its own ≥3-draw reference set.
4. **Corpus/judge pinning.** Probes use the frozen corpus and record the judge model; cross-judge comparisons are
   forbidden (`SCORING_SPEC.md:219-234`: regimes not comparable).

**Audit procedure.** Probe manifest must name the champion config hash and cite its reference draws; an
empty-AC run is diffed against the pinned champion config before any wave's probe is accepted.

**Stop.** Any probe without the champion arm, or any shipped wave that cannot reproduce the champion under
empty-AC, is reverted.
