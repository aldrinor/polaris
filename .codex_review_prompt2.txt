## 1. Executive summary

The codebase earns a high accuracy score but presents to an independent reviewer as **high-signal-but-untended**: the engineering is real, the discipline around it is not visible or not enforced. Four things a Telus reviewer will flag in the first hour:

1. **No enforced Python quality gate (HIGH).** A 1,082-file pytest suite exists but **is never run in CI**; there is no lint, no type-check, no coverage gate, and **no `pyproject.toml` anywhere**. Typing is actually excellent (96% return-annotated) but undefended. To a reviewer, "has tests but doesn't gate them" scores the same as "has no tests." All 28 CI `uses:` are on mutable tags — a live supply-chain finding.

2. **No typed central config; ~900 undocumented env knobs (HIGH).** 1,707 `os.getenv` sites resolve ~923 distinct keys; `.env.example` documents only 24. Model slugs, temperatures, and thresholds are hardcoded as module constants across ≥9 modules, and 21 hand-rolled `_env_*` helpers are copy-pasted. 3. **Marketing-adjective and version-marker naming on the critical path (HIGH).** A **live 3-way `graph.py` / `graph_v2.py` / `graph_v3.py` fork** routed by a `PG_GRAPH_VERSION` env var, confirmed dead in-tree twins (`report_assembler_v2.py`, `honest_sweep_integration.py`, both zero importers), and `honest_*` adjectives sitting **on the release-scoring path** a reviewer scrutinizes first.

4. **A checkpoint that is built but silently inert (MEDIUM, but embarrassing).** `save_generation_snapshot` is defined and never called anywhere; its loader is fully wired. The generation-draft resume capability is 90% present and dead on the write side, so every resume re-bills the ~30-40min generation.

**Headline of the plan:** Fix the *cheap, high-visibility, zero-pipeline-risk* items first (secret/attribution hygiene, delete dead twins, stand up `pyproject.toml` + informational CI, README/architecture refresh), then do the *mechanical-but-guarded* work (config extraction defaulting byte-identically to today's literals; renames behind deprecation aliases; wire the dead checkpoint), and gate the pipeline-touching changes (the graph-fork resolution, the coverage/type ratchet-to-required) behind a full score-parity replay. **De-vendoring is largely a branding/attribution exercise, not a legal violation** — the flagship methods are verified clean-room reimplementations.

## 4. Guardrails (apply to every commit)

1. **Byte-identical-by-construction for config.** Every extracted config value **defaults to the exact current hardcoded literal** and keeps the identical `PG_*` env name. A golden test must assert `Settings()` reproduces every current default before any module migrates.
2. **No behavior change in a rename/extraction commit.** Renames and config extraction are pure mechanical moves. A threshold/default/slug change never rides along in the same commit — it is a separate, separately-reviewed PR.
3. **Renames go through aliases, not find-replace.** Symbols and filenames are renamed; `PG_*` env-var string literals and enum/stage string values are preserved (or aliased). One rename per commit, each with an acceptance-harness before/after.
4. **DATA-only checkpoint invariant.** Any snapshot persists pre-verification data only (`raw_drafts`, outline, atom/claim catalog) — never `verified_text`, never a stored verdict. No faithfulness gate is ever skipped by replaying a stored decision. Writes are atomic + fail-open; resume behavior is OFF by default.
5. **Acceptance harness is the arbiter.** `acceptance_outline_agent.py` runs before and after every phase touching importable code; anything touching graph selection additionally requires a full replay/score-parity run.
6. **CI lands informational, ratchets to required.** No new check is a required status check until one green baseline exists on `main`. Thresholds start at the measured floor and ratchet upward.
7. **`third_party/` legal fixes are decoupled** — they add files under a tree the runtime never imports and may land anytime with zero coordination.
8. **Do not rename legitimate domain terms:** `SmartArt*`, `deep_crawl`, `deepen_evidence`, `finalize_body`, and the external service names OpenAlex/PubMed/Semantic Scholar. Keep paper citations in docstrings.
