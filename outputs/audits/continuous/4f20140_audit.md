## P3

**P3.1 — F-26 default selection + first-Tab UX.** Native `radiogroup` ARIA pattern usually pairs with arrow-key cycling (Up/Down/Left/Right move between radios) rather than Tab cycling. The F-28 keyboard test (`accessibility.spec.ts:244-263`) uses `radios.nth(1).focus()` directly + Space; doesn't validate arrow-key behavior. WAI-ARIA Authoring Practices says radiogroup MUST handle arrow keys to cycle (and Tab moves OUT of the group, not within). With each `<button role="radio">` keeping its own `tabindex=0` (the default for `<button>`), Tab moves through every radio one-by-one — which works but isn't the ARIA-conformant pattern. Not a security issue (the form behavior is correct), and not strictly a WCAG-AA failure. **Tag: out-of-scope-observed (a11y polish for cycle-12).**

**P3.2 — F-28 sweep covers chromium only.** The sweep test runs only against `chromium` per `playwright.config.ts` defaults. Mobile-touch/Safari/Firefox target-size rendering can differ slightly. Not a security issue and out of cycle-10's scope. **Tag: out-of-scope-observed.**
