# Codex M-61 audit

**Verdict**: REJECT

## Answers

1. Structured provenance (rev #6): The 8 fields are enumerated in `StructuredProvenance` and required by `_parse_provenance` (`src/polaris_graph/retrieval/human_gap_completion.py:93-131`, `:225-290`). That is the right minimum, but it is not fully sufficient as shipped: extra legacy fields are still silently accepted, several fields are only validated as non-empty strings, and the provenance is later dropped at the `FrameRow` boundary. I would not remove any of the 8. I would add `other_justification` when `source_type=="other"` and persist the provenance itself, or at least a stable provenance id, downstream.
2. source_type allowlist: `other` without a required justification is too loose. It should require an explicit justification field at parse time; otherwise the pressure valve becomes an unaudited bucket.
3. artifact_sha256 validation: Sufficient for tamper-evident audit replay. The value is normalized to lowercase, regex-checked, and independently recomputable via `compute_artifact_sha256()` (`src/polaris_graph/retrieval/human_gap_completion.py:255-261`, `:462-467`). It is not sufficient for authenticity by itself; it only binds the retained artifact.
4. acquired_at validation: Requiring timezone-aware ISO-8601 is the right baseline. The implementation is less strict than the docstring: it accepts any timezone offset, not UTC only (`src/polaris_graph/retrieval/human_gap_completion.py:109-110`, `:263-279`). If UTC is part of the contract, add an explicit zero-offset check.
5. DOI substitution attack: Not sufficient. `validate_against_tasks()` only rejects mismatches when both DOIs are truthy; `task.doi=<real DOI>` plus `completion.doi=None` is accepted, so an operator can bypass the paper-binding check by omission (`src/polaris_graph/retrieval/human_gap_completion.py:364-373`). I confirmed that path executes.
6. Unmatched entity_id: Yes. Completions for entities absent from the M-60 task list are rejected with an explicit reason (`src/polaris_graph/retrieval/human_gap_completion.py:355-363`). That part does prevent off-list completions.
7. Human-curated permanent flag (rev #6): No. `quote_source` is set to `"human_curated"`, but `FrameRow.provenance_class` is written as `ProvenanceClass.ABSTRACT_ONLY`, so the flag is not permanent across downstream handoffs (`src/polaris_graph/retrieval/human_gap_completion.py:401-450`). The test suite currently locks in that downgrade (`tests/polaris_graph/test_m61_human_completion.py:342-344`).
8. Provenance object not in FrameRow: Wrong boundary as shipped. Only `artifact_retention_path` survives via `url`; the other seven provenance fields are dropped (`src/polaris_graph/retrieval/human_gap_completion.py:412-416`, `:437-458`). An in-memory map keyed only by `entity_id` is brittle, and `validate_against_tasks()` does not enforce one completion per entity, so the map can become ambiguous (`src/polaris_graph/retrieval/human_gap_completion.py:349-377`). `FrameRow` should carry structured provenance or a stable provenance pointer/id.
9. strict_verify division of concerns (rev #7): The conceptual split is documented clearly enough in the module docstring and `to_frame_rows()` notes (`src/polaris_graph/retrieval/human_gap_completion.py:35-55`, `:418-422`). The issue is operational: the authenticity evidence is not durably threaded through the row boundary.
10. Methods disclosure: Incomplete and partly inaccurate. Separate counts are good, but the function tells readers to inspect manifest entries with `provenance_class='human_curated'` and curator attribution + `artifact_sha256`, while this module never emits that provenance_class and never serializes those fields (`src/polaris_graph/retrieval/human_gap_completion.py:470-485`). It also says "licensed sources" even though allowed `source_type` values include `author_communication` and `other`.
11. Entity-type-agnostic (rev #7): Yes within this module. There is no entity-type branching, and `doi=None` covers non-DOI entities; the tests include statute and DFT (`src/polaris_graph/retrieval/human_gap_completion.py:56-59`, `:138-146`; `tests/polaris_graph/test_m61_human_completion.py:384-405`).
12. Determinism: Correctly scoped. `parse_completion`, `_parse_provenance`, `validate_against_tasks`, and `to_frame_rows` are pure given inputs; `load_completions()` is the I/O boundary (`src/polaris_graph/retrieval/human_gap_completion.py:61-65`, `:163-467`).

## Findings

- Blocker: DOI omission bypasses the claimed paper-binding fraud defense. `validate_against_tasks()` accepts `task.doi=<real DOI>` with `completion.doi=None`, so an operator can evade the mismatch gate by leaving DOI blank. `src/polaris_graph/retrieval/human_gap_completion.py:364-373`; coverage gap in `tests/polaris_graph/test_m61_human_completion.py:284-313`.
- Blocker: The permanent `human_curated` flag is not actually permanent. `to_frame_rows()` writes `quote_source="human_curated"` but stores `provenance_class=ProvenanceClass.ABSTRACT_ONLY`, while `compose_methods_disclosure_human_curated()` claims downstream manifest entries will have `provenance_class='human_curated'`. Those two claims cannot both be true. `src/polaris_graph/retrieval/human_gap_completion.py:401-450`, `:470-485`; test bakes in the downgrade at `tests/polaris_graph/test_m61_human_completion.py:342-344`.
- Blocker: Structured provenance does not survive the `FrameRow` handoff. Only `artifact_retention_path` is threaded through `url`; `curator_id`, `source_type`, `source_locator`, `acquired_at`, `artifact_sha256`, `quote_page_range`, and `attestation` are dropped. The documented fallback "in-memory map keyed by entity_id" is lossy because multiple completions per entity are not rejected. `src/polaris_graph/retrieval/human_gap_completion.py:349-377`, `:412-416`, `:437-458`.
- Medium: The schema is not actually closed against legacy free-text channels. A record with the required `provenance` object plus extra `consent_proof` or other undeclared keys still parses; that is not "free-text-only acceptance", but it is still silent acceptance of a legacy free-text field. `src/polaris_graph/retrieval/human_gap_completion.py:163-222`, `:225-290`.
- Medium: `source_type="other"` has no required justification, which weakens auditability exactly where the allowlist is meant to constrain it. `src/polaris_graph/retrieval/human_gap_completion.py:152-158`, `:247-253`.
- Nit: `acquired_at` is documented as UTC but the implementation accepts any timezone-aware offset. `src/polaris_graph/retrieval/human_gap_completion.py:109-110`, `:263-279`.
