**Verdict**: CONDITIONAL-blockers

## Answers

1. Root-cause vs band-aid: M-66a is diagnostic plus tactical, not root-cause as written. Run-2 evidence shows SURPASS-6 died because the slot payload was mostly `not extractable from available primary content`, then every sentence failed `strict_verify` overlap; regrouping in `run_contract_section` is a plausible secondary failure mode, but not the main one. Keep the telemetry, but do not treat overlap loosening as the primary fix unless a post-M-66b rerun still shows valid extracted fields being dropped.
2. Fix sequencing: M-66c is independent and can land anytime. M-66b should land before any semantic loosening in M-66a; split M-66a into `telemetry first` and `verifier relaxation only if still needed`.
3. M-66b scope: For FDA/EMA/NICE/HC public pages, automated `url_pattern` fetch belongs in M-56, not M-61. M-61 is the fallback for paywalled or licensed material like SURPASS-CVOT, not routine public regulatory provisioning; the scope creep is real, so `url_pattern` fetch and OA full-text trial fetch need separate acceptance/tests.
4. Acceptance criteria: Not strong enough. The efficacy gate is internally inconsistent (`SURPASS-1/2/3/4/5/6 + SURMOUNT-2 + SURPASS-CVOT` is 8, not 7), and row-count-only table/timeline gates can still pass malformed output. Add gates on actual `frame_coverage_report` pass states, exact rendered subsection accounting, semantic table/timeline integrity, and false `7/7 topics covered` prevention.
5. Dimensional impact honesty: `5 BB + 2 BO + 0 LB` is optimistic. Regulatory and Jurisdiction are partly the same lift from M-66b, Structure does not get to BB without an explicit table/timeline fix, and Narrative depth should not be pre-booked from source volume alone. A more honest pre-run projection is closer to `2-3 BB, 3-4 BO, 1 LB`.
6. Narrative depth projection: Mostly a hand-wave. More fetched content helps slot extraction, but dim 7 comes from synthesis behavior in Safety/Comparative and cross-trial interpretation, not just quote length; expect BO only if prompt/rendering changes also deepen synthesis.
7. Thomas clamp: The yaml realignment is the real fix. Do not expand M-66 to “Thomas full paper” unless the article is trivially reachable through the same OA path and there is a specific dim-7 reason; clamp is a bounded slot, not the main ship blocker.
8. Regression risk: The `$10/run` ceiling is still safe; run-2 cost was only `$0.00449`. The real risk is prompt bloat and noisier extraction if `direct_quote` jumps from short abstracts to 25K-char blobs, so cap by section/window and add regression coverage for token growth and precision.
9. Omitted blockers: SURPASS-CVOT remains unresolved and cannot stay inside a hard “all efficacy subsections” gate unless it moves to M-61/manual completion. Trial Summary and Timeline malformation also lack a dedicated fix, and the false `7/7 topics covered` claim still needs an explicit guard.
10. Ship criteria: If run-3 truly reaches `BEAT-BOTH >=5/7`, has no LB, and closes the false-pass conditions above, that is a valid V30 Phase-2 ship. It is still ship-and-continue: V31/V32 likely still need narrative-depth calibration, CVOT/paywall policy, and stronger table/timeline fidelity.
