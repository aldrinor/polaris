### 3.4 Scenario-family pairwise relations + pairwise DEFF

**Pairwise relation rules** (unchanged from v3.2): two claims are scenario-family-related iff ANY of criteria 1-5 (same report, evidence Jaccard≥0.5, same template/SME/24h, same prompt family, microtopic+stratum+correlation).

**Pairwise DEFF formula**:

```
DEFF = 1 + 2 × P × ρ / N
  N = total claims in stratum
  P = total unordered pairs of related claims in stratum (deduplicated; see below)
  ρ = ICC ceiling per §4.4

Edge cases (Codex round-3 confirmed):
  P=0 → DEFF=1
  P=N(N-1)/2 (saturation) → DEFF = 1 + (N-1)ρ (exchangeable cluster)
  saturation does not break the formula; it just collapses n_eff

Mitigations:
  - **Pair deduplication required** (NEW v3.3 per Codex round-3): if a pair
    satisfies multiple criteria, it counts as ONE related pair, not multiple.
    Implementation: deterministic OR over criteria, stored as boolean per pair.
  - ρ=0.20 sensitivity retained (covers missed positive correlations among
    "unrelated" pairs and true pair correlations above ICC ceiling).
```

**Audit prerequisite**: max claim degree, 95th-percentile degree — committed pre-Phase-0b. 95th-percentile > 25 triggers refinement.

**Pairwise lookup table**: (claim_id_i, claim_id_j) → related? boolean, hash-pinned.

### 4.4 ICC justification protocol (REVISED v3.3 per Codex round-3 §4)

**Pre-designated surrogates** (unchanged from v3.2): 4 surrogates (source-disagreement primary; evidence-completeness, retrieval-tier, verifier escalation rate as secondaries).

**ICC estimator** (REVISED v3.3 per Codex round-3 §4):

```
PRIMARY: beta-binomial / logistic-normal random-intercept GLMM where
  denominators are available (statsmodels.BinomialBayesMixedGLM or
  glmmTMB equivalent in R).
  
  ICC computed via Nakagawa-Schielzeth distribution-specific variance for
  binomial GLMM (Nakagawa, Johnson, Schielzeth 2017).
  
  Locked random seed for any MCMC/variational inference.

SENSITIVITY / CONVERGENCE FALLBACK: empirical-logit weighted ANOVA on
  per-customer proportions with:
    - Continuity correction +0.5/+1 for boundary proportions (Haldane-Anscombe)
    - Inverse-variance weighting by denominator
    - Zero-denominator customers dropped from ICC computation, reported separately
    - Boundary ICC clipped to [0, 1]
  
  Used IFF GLMM fails to converge OR per pre-registered convergence diagnostics.

CI: parametric bootstrap on logit-ICC (B=10000) for either estimator.
```

**Why beta-binomial GLMM primary** (per Codex round-3): canonical for count-over-denominator proportions with unequal denominators and overdispersion. Logit-ANOVA on raw proportions is heteroskedastic; viable as fallback only.

**Escalation rule** (REVISED v3.3 per Codex round-3 §3 — added tolerance + cap):

```
If any surrogate ICC point estimate > 0.10 + 1e-12
    OR any one-sided upper 95% CI > 0.15 + 1e-12:
        Escalation triggered.
        
        Escalated ICC ceiling = min(
            max(
                round_up_to_nearest(max_observed_point_estimate, 0.05),
                round_up_to_nearest(max_observed_upper_95_CI, 0.05),
                0.15
            ),
            1.0   # capped at 1.0, NEW v3.3 per Codex round-3
        )
        
        round_up_to_nearest(x, 0.05) = ceil(x / 0.05) × 0.05
        Strict comparison after deterministic rounding (1e-12 tolerance)

§P4 Category-3 amendment filed.
```

### 6.4 Prevalence-standardized precision + PPV CI (REVISED v3.3 per Codex round-3 §6)

**PPV CI propagation** (REVISED v3.3 per Codex round-3 §6):

```
PPV = sens × prev / [sens × prev + (1 − spec) × (1 − prev)]

Bootstrap method:
  - If sens/spec and canary prevalence come from THE SAME sample units:
      Joint cluster bootstrap (B=10000) preserving customer-cluster structure.
      Compute PPV per replicate from joint (sens, spec, prev) realization.
  
  - If sens/spec and canary prevalence come from SEPARATE samples
      (e.g., gold-set for sens/spec, canary for prev):
      INDEPENDENT bootstrap of each source, then combine per replicate.
      Preserve cluster structure within each source.

To avoid impossible PPV values (>1 or <0):
  Use logit-scale draws for sens/spec/prev, then back-transform; OR
  Constrained beta/binomial draws with informative-conjugate priors at the
  source-distribution scale (NOT for the PPV gate itself — keep frequentist).

Random seeds hash-pinned per cycle.
```
