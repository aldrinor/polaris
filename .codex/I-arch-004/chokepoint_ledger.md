## 5) error_unexpected Root Cause + Graceful-GAP Fix

**Root cause:** `multi_section_generator.py:5509 & 5550` (+ M-50 `:6432`) run all sections in a single `asyncio.gather` **without `return_exceptions=True`**. When one section's wall-clock guard raises `TimeoutError` (`:93`), the gather fails, all sibling sections are cancelled, and the exception propagates to the bare `except Exception` at `run_honest_sweep_r3.py:8484`. There, `_credibility_abort_status(exc)` returns None (a `TimeoutError` is not a `CredibilityPassError`), so it routes to `error_unexpected`/status `error` (`:8500-8503`) and writes only a content-free error manifest. The whole 3-hour run dies on one slow section, with no per-section isolation and no checkpoint.

**The fix is necessarily TWO parts — both must land** (`return_exceptions` catches exceptions, not *hangs*; a 0-socket wedge with no wall-clock = infinite gather hang, worse than the crash):

- **(a) Right-sized wall-clock** so the guard fires only on a genuine wedge, never on legitimate long reasoning: `PG_SECTION_WALLCLOCK_SECONDS` calculated per §2 (~11000-18000s), and `run_gate_b.py` must **slate it + FAIL LOUD in preflight if it resolves below the floor** — this is what permanently stops smoke-value (600) inheritance. This converts a wedge into a clean `TimeoutError` instead of an infinite hang.
- **(b) `return_exceptions=True` on the three primary gathers + map the exception to a gap-stub `SectionResult`** (the stub path ~`321-347`), not a raw re-raise. One section's `TimeoutError` becomes a **GAP in the report** (one missing section, the run completes with the other 5), not a whole-run `error_unexpected`. Combined with the per-section checkpoint (§3), a wedged section degrades to a one-section retry/gap. (Note: the existing regen gather at `:5846` already has `return_exceptions=True` but re-raises `CredibilityPassError` at `:5856` — keep that fail-loud carve-out; faithfulness coverage gaps must still abort, only the section-timeout becomes a gap.)
