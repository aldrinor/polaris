## P1-2: emit_event non-raising

Resolution:
```python
def emit_event(external_run_id, event_type, payload, *, redis_client=None):
    if not external_run_id:
        return  # CLI sweep — no v6 stream
    try:
        client = redis_client or _get_sync_redis()
        client.xadd(EVENT_STREAM_KEY.format(run_id=external_run_id),
                    {"event_type": event_type, "payload": json.dumps(payload)},
                    maxlen=10000, approximate=True)
    except (redis.exceptions.RedisError, ConnectionError) as exc:
        logger.warning("[run_events] emit failed run_id=%s event_type=%s: %s",
                       external_run_id, event_type, exc)
    except Exception as exc:  # noqa: BLE001 — observability must NEVER block pipeline
        logger.warning("[run_events] emit unexpected error: %s", exc)
```
Pipeline-A `error_unexpected` cannot be caused by Redis observability.

## P1-3: stream_unavailable only on Redis connection failure

Resolution: SSE reader checks run_store first when stream is empty:
```python
async for event in _redis_stream_source(run_id, last_id):
    yield event
# When XREAD raises ConnectionError → terminal stream_unavailable
# When XREAD returns empty for 5s window → keepalive ping, NOT terminal
# Only after run_store shows run finished AND no more events for 10s → close cleanly
```

Empty-stream-while-queued/in_progress → keepalive forever (or until client disconnects), per `run_store.get_run(run_id).lifecycle_status`. Empty after `lifecycle_status == completed/failed` AND no terminal event seen for 10s → emit synthetic `run_complete(status=stream_lost)` and close (degraded but honest).
