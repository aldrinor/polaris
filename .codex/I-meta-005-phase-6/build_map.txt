SUMMARY: POLARIS emits 1500–3000-word multi-section verified reports via three-stage architecture:

**Stage 1 (OUTLINE):** LLM or planner emits section titles + focus statements + evidence allocations. ON-mode (research_plan provided): section titles + archetype tags + count are FIXED by pre-retrieved, SHA-pinned ResearchPlan.outline; the code only assigns evidence. OFF-mode (legacy): LLM call via _call_outline emits JSON section plan, fallback to deterministic 3-section when invalid.

**Stage 2 (PER-SECTION GENERATION):** Each section runs async _run_section: (a) draft prose with [ev_XXX:start-end] provenance tokens (b) rewrite to fix token spans (c) strict_verify: checks evidence ID existence, span existence in direct_quote, numeric value presence in span; drops unverified sentences as core filter (d) resolve_provenance_to_citations: strip tokens, renumber citations [1] [2]... globally.

**Stage 3 (ASSEMBLY & POST-SECTIONS):** Merge bibliographies, remap citation numbers across sections. Then optional appenders (UNVERIFIED): Limitations prose (R-1, LLM-synthesized from telemetry only, no [ev] tokens), analyst_synthesis (I-bug-105, LLM-written expert commentary over verified+biblio, explicitly NOT span-verified, clearly labeled in report), M-42b trial table/timeline (deterministic builder from direct_quote, then LLM fallback), M-50 per-trial subsections. All appenders are DISABLED in partial_mode.

- Receives: verified_prose_joined (all sections concatenated with ##headers), global_biblio, evidence_rows (full pool), research_question
- Output: 1500–3000 words of expert interpretation
- UNVERIFIED: no [ev:...] tokens, cites by bibliography [N] only, explicitly NOT span-verified

- Omitted when empty (no disclosure block per Codex iter-1)
