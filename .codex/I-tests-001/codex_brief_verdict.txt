## Root cause analysis

**ALL 65 errors share one root cause:** `ModuleNotFoundError: No module named 'polaris_graph'`.

The POLARIS codebase has two coexisting import-prefix conventions:

| Convention | Used by | Notes |
|---|---|---|
| `from polaris_graph.X import Y` (NO `src.` prefix) | Most tests + some `src/` modules (`generator2/strict_verify.py`, `generator2/provenance.py`, etc.) | Works when `src/` is on `sys.path` |
| `from src.polaris_graph.X import Y` | Other tests + most `scripts/` + `src/polaris_graph/llm/openrouter_client.py` | Works when repo root is on `sys.path` |

Pytest's default `sys.path` derivation puts the repo root on path but NOT `src/`. So:
- `from src.polaris_graph.X import Y` → resolves
- `from polaris_graph.X import Y` → `ModuleNotFoundError`

The 65 affected test files import via the `polaris_graph.X` convention.

## Recommended fix

Single root-cause fix unblocks all 65:

**Add a top-level `conftest.py` (or update `pytest.ini`) that prepends both `src/` and the repo root to `sys.path`.** The shape:

```python
# tests/conftest.py (new file)
import sys
from pathlib import Path

_REPO_ROOT = Path(__file__).resolve().parent.parent
_SRC = _REPO_ROOT / "src"
for p in (_REPO_ROOT, _SRC):
    if str(p) not in sys.path:
        sys.path.insert(0, str(p))
```

After this fix, both `from polaris_graph.X import Y` and `from src.polaris_graph.X import Y` resolve to the SAME module instance (per the `src/` package layout — both spellings point at the same files).

**Caveat — Python module-instance identity:** even with both on path, the two import spellings load as **separate sys.modules entries** with **separate ContextVar instances** (verified empirically in I-bug-100 PR #379). This is acceptable for resolving collection errors, but tests that share state across the two spellings could still see drift. The mitigation: I-bug-100 already standardized on `src.polaris_graph.llm.openrouter_client` for the cost-tracking ContextVars; future modules with shared state should follow the same convention.

## Why this is OUT OF SCOPE for I-tests-001

The acceptance criterion was "per-test classification + per-class fix plan". Both delivered above. Implementing the fix is a separate concern with its own risk surface (re-running 3547 tests after sys.path change to confirm no regression in currently-passing tests).

A follow-up issue should:
1. Add the conftest.py sys.path fix.
2. Re-run the full suite to confirm 65 → 0 collection errors AND no new failures.
3. Document the canonical import convention as `from src.polaris_graph.X import Y` in CLAUDE.md or AGENTS.md so future tests don't regress.
