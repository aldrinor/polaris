## The core finding (deeper than "test harness pages")

POLARIS's features were each built as **substrates + test-harness pages + golden
fixtures**, and each GitHub issue was completed *as scoped* and Codex-reviewed.
BUT the **rich product surfaces do not consume live run output** — the inspector,
charts, follow-up, compare, bundle paths are **fixture-bound** (they display
golden test fixtures, not the artifacts a real run produces).

Consequence: even with a working LLM, a real run will produce a run — but
click-through inspection, chart provenance, follow-up, compare, pin replay,
memory, and bundle export **from that same run** are not wired. This is the
`bpei_phantom_completion` pattern: issue-level "done" ≠ product-level "done."

So the GPU-wait window is NOT "testing time." It is **integration time** — and
that work mostly does not need the GPU.
