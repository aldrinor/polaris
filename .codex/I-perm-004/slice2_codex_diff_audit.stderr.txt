## THE safety property (only real P0 class here): the bar is NEVER lowered
The §-1.1-lethal failure is laundering a drop into a pass. Verify against the code:
- The resolver's `judge_fn` is the closure `_candidate_passes(_sentence, span, _span_text)` which RE-BINDS the token to `span` and runs `verify_sentence_provenance(..., allow_local_window_fallback=False).is_verified` — the SAME full gate the first-passing loop uses. So an accepted span ALWAYS passes the exact content+numeric+entailment bar.
- After argmax picks `best`, the code re-binds to `best.best_span` and verifies ONCE MORE; `if not v.is_verified: return None` (defensive — never launder).
- The argmax can only CHOOSE among judge-accepted candidates; it changes WHICH supporting span is bound, never WHETHER an unsupported claim passes.
- OFF (`PG_SPAN_RESOLVER` unset) -> the new branch is skipped entirely; the first-passing loop is byte-identical (the existing 13 reanchor tests + the telemetry-contract test pass with the new counter=0).

If you can show a path where flag-ON accepts a sentence that flag-OFF would have DROPPED (i.e. the argmax recovers something first-passing would not, AND that something is not genuinely gate-passing), that is a P0. Note: argmax judging only `top_k` candidates means it could recover FEWER than the exhaustive first-passing loop (a SAFE reduction), or a DIFFERENT (better-quality) span among passers — but never a span the gate rejects.

  116: def classify_span(text: str) -> str:
  117:     """Deterministically classify a candidate span's TEXT into a ``provenance_quality`` bucket.
  118: 
  119:     Boilerplate patterns (url / altmetric / reference-list / affiliation / nav) take precedence;
  120:     otherwise a short, terminator-free fragment is a title/header and a longer sentence-shaped
  121:     fragment is prose. Never raises; empty/whitespace -> ``header`` (uninformative boilerplate).

  123:     stripped = (text or "").strip()

  127:     # so the URL bucket is checked LAST among boilerplate).
  128:     if _ALTMETRIC_RE.search(stripped):
  129:         return QUALITY_ALTMETRIC
  130:     if _NAV_RE.search(stripped) or len(_NAV_SEPARATOR_RE.findall(stripped)) >= 2:
  131:         return QUALITY_NAV_LINK
  132:     if _REFERENCE_RE.search(stripped):

  134:     if _AFFILIATION_RE.search(stripped):
  135:         return QUALITY_AFFILIATION
  136:     words = _WORD_RE.findall(stripped)
  137:     if _URL_RE.search(stripped):

  139:         # bulk IS the URL is url-boilerplate. Heuristic: URL match AND few sentence words / mostly
  140:         # non-letters.
  141:         letters = sum(c.isalpha() for c in stripped)
  142:         if len(words) < _PROSE_MIN_WORDS or letters < len(stripped) * 0.55:
  143:             return QUALITY_URL

  157:     if len(stripped) <= _TITLE_MAX_CHARS and not has_terminator:
  158:         return QUALITY_HEADER if stripped.endswith(":") or len(words) <= 3 else QUALITY_TITLE
  159:     return QUALITY_PROSE

  175: class SpanResolution:
  176:     """The argmax entailing span for one claim (or ``None`` when nothing in-row entails)."""
  177: 
  178:     best_span: tuple[int, int]

  180:     provenance_quality: str

  182:     entailed: bool

  184: 
  185: # Entailment-dominated confidence base, then subtract the boilerplate penalty and (optionally) a
  186: # numeric-mismatch penalty, then add a small lexical-overlap nudge that can never by itself promote
  187: # a span. Clamped to [0, 1]. Tuned so prose-entailed >= ~0.7 (moderate/high) and boilerplate-entailed
  188: # <= ~0.55 (low), keeping the §-1.1 invariant that a title-only-supported claim is never high.
  189: _ENTAILED_BASE = 1.0
  190: _NUMERIC_MISMATCH_PENALTY = 0.25
  191: _LEXICAL_NUDGE = 0.05

  194: def _span_confidence(
  195:     quality: str,
  196:     lexical: float,
  197:     *,
  198:     numeric_ok: bool,
  199: ) -> float:
  200:     confidence = _ENTAILED_BASE - _QUALITY_PENALTY.get(quality, 0.6)
  201:     if not numeric_ok:
  202:         confidence -= _NUMERIC_MISMATCH_PENALTY
  203:     confidence += _LEXICAL_NUDGE * max(0.0, min(1.0, lexical))
  204:     return max(0.0, min(1.0, confidence))

  207: def resolve_best_entailing_span(
  208:     direct_quote: str,
  209:     sentence: str,
  210:     candidate_spans: list[tuple[int, int]],
  211:     *,
  212:     judge_fn: Callable[[str, tuple[int, int], str], bool],
  213:     numeric_match_fn: Optional[Callable[[str, str], bool]] = None,
  214:     top_k: int = 4,
  215: ) -> Optional[SpanResolution]:
  216:     """Choose the best genuinely-ENTAILING span for ``sentence`` within ``direct_quote``.
  217: 
  218:     ``candidate_spans`` are ``(start, end)`` slices of ``direct_quote`` (caller supplies them, e.g.
  219:     via ``provenance_generator._reanchor_candidate_spans``). The candidates are PRE-ranked cheaply by
  220:     (prose-first, lexical overlap); only the top ``top_k`` are handed to ``judge_fn`` (bounded judge
  221:     calls). ``judge_fn(sentence, span, span_text) -> bool`` is the BINDING gate — the wiring passes a
  222:     closure that RE-BINDS the [#ev] token to ``span`` and runs the SAME full faithfulness gate
  223:     (content + numeric + entailment, ``allow_local_window_fallback=False``), so the resolver returns
  224:     ONLY a span the gate accepted and a non-entailing span can never be manufactured into support.
  225:     The span OFFSETS are passed (not just the text) precisely so the judge can re-bind. Among
  226:     accepted candidates the ARGMAX prefers higher provenance quality (prose over boilerplate) then
  227:     higher confidence. Returns ``None`` when no candidate is accepted (caller keeps the drop /
  228:     labels "no grounded source").
  229: 
  230:     ``numeric_match_fn(sentence, span_text) -> bool`` (optional) lets the caller require the
  231:     sentence's numbers to appear verbatim in the chosen span; a mismatch lowers confidence (never
  232:     promotes). When omitted, numeric agreement is treated as satisfied (no adjustment).
  233:     """
  234:     n = len(direct_quote or "")
  235:     # Each entry: (prerank_key, (start, end), span_text, quality, lexical).
  236:     scored: list[tuple[tuple[float, float], tuple[int, int], str, str, float]] = []
  237:     for start, end in candidate_spans:
  238:         if not (0 <= start < end <= n):
  239:             continue
  240:         span_text = direct_quote[start:end]
  241:         if not span_text.strip():
  242:             continue
  243:         quality = classify_span(span_text)
  244:         lexical = _lexical_overlap(sentence, span_text)
  245:         # PRE-rank key: prose first (smallest penalty), then lexical overlap. This only orders which
  246:         # top_k reach the judge; it never decides entailment.
  247:         prerank = (-_QUALITY_PENALTY.get(quality, 0.6), lexical)
  248:         scored.append((prerank, (start, end), span_text, quality, lexical))
  249: 
  250:     # Pre-rank descending and judge ONLY the top_k (bounded judge calls).
  251:     scored.sort(key=lambda r: r[0], reverse=True)
  252:     best: Optional[SpanResolution] = None
  253:     best_key: tuple[float, float] = (-1.0, -1.0)
  254:     for _prerank, span, span_text, quality, lexical in scored[: max(1, top_k)]:
  255:         if not judge_fn(sentence, span, span_text):
  256:             continue
  257:         numeric_ok = True if numeric_match_fn is None else numeric_match_fn(sentence, span_text)
  258:         confidence = _span_confidence(quality, lexical, numeric_ok=numeric_ok)
  259:         # ARGMAX: prefer best quality (smallest penalty), then highest confidence.
  260:         key = (-_QUALITY_PENALTY.get(quality, 0.6), confidence)
  261:         if key > best_key:
  262:             best_key = key
  263:             best = SpanResolution(
  264:                 best_span=span,
  265:                 span_text=span_text,
  266:                 provenance_quality=quality,
  267:                 confidence=confidence,
  268:                 entailed=True,
  269:             )
  270:     return best

 1002: def _reanchor_candidate_spans(direct_quote: str) -> list[tuple[int, int]]:
 1003:     """Enumerate a BOUNDED set of candidate (start, end) spans inside a row's
 1004:     ``direct_quote``.
 1005: 
 1006:     Candidates are produced by (a) DECIMAL-AWARE sentence-segmenting the row text
 1007:     and (b) a sliding window of PG_PROVENANCE_REANCHOR_WINDOW chars stepped by
 1008:     half the window. Both are clamped to PG_PROVENANCE_REANCHOR_MAX_CANDIDATES
 1009:     total so there is NO compute blow-up on a large row. Each span is a valid
 1010:     ``0 <= start < end <= len(direct_quote)`` slice, so the reused verifier's
 1011:     span-bounds checks pass naturally.
 1012: 
 1013:     I-complete-003 iter-2 (#1189) P2-1:
 1014:       * segmentation does NOT split a decimal number at its period (a period
 1015:         between two digits is a decimal point, not a terminator), so a sentence
 1016:         like ``...reduction of 1.5 percent.`` stays in ONE candidate segment;
 1017:       * the DEGENERATE whole-row sliding-window candidate is SUPPRESSED for rows
 1018:         shorter than the window — re-binding the token to the WHOLE row weakens
 1019:         the claimed citation precision (it is not a tight supporting span). A
 1020:         sentence SEGMENT that legitimately spans the entire row (a single-sentence
 1021:         verbatim lift) is still emitted by branch (a), so genuine recovery on a
 1022:         one-sentence row is preserved.
 1023:     """
 1024:     if not direct_quote:
 1025:         return []
 1026:     n = len(direct_quote)
 1027:     cap = max(1, PG_PROVENANCE_REANCHOR_MAX_CANDIDATES)
 1028:     seen: set[tuple[int, int]] = set()
 1029:     candidates: list[tuple[int, int]] = []
 1030: 
 1031:     def _add(start: int, end: int) -> None:
 1032:         start = max(0, start)
 1033:         end = min(n, end)
 1034:         if start >= end:
 1035:             return
 1036:         key = (start, end)
 1037:         if key in seen:
 1038:             return
 1039:         seen.add(key)
 1040:         candidates.append(key)
 1041: 
 1042:     # (a) DECIMAL-AWARE sentence-segment candidates: split on sentence
 1043:     # terminators that are NOT decimal points, keeping the trailing terminator
 1044:     # with its segment. Each non-blank segment becomes a candidate span.
 1045:     seg_start = 0
 1046:     for m in _REANCHOR_SENTENCE_TERMINATOR_RE.finditer(direct_quote):
 1047:         if len(candidates) >= cap:
 1048:             return candidates[:cap]
 1049:         seg_end = m.end()  # include the terminator char in the segment
 1050:         if direct_quote[seg_start:seg_end].strip():
 1051:             _add(seg_start, seg_end)
 1052:         seg_start = seg_end
 1053:     # Trailing segment (no terminating punctuation at end of row).
 1054:     if seg_start < n and direct_quote[seg_start:n].strip() and len(candidates) < cap:
 1055:         _add(seg_start, n)
 1056: 
 1057:     # (b) Sliding-window candidates (half-window step) over the full row, to
 1058:     # catch support that straddles sentence boundaries. P2-1: SKIP the degenerate
 1059:     # whole-row window — when the window is >= the row length the only candidate
 1060:     # the sliding loop would add is (0, n), a whole-row rebind that weakens
 1061:     # citation precision. Branch (a) already supplies any legitimate full-row
 1062:     # SENTENCE segment, so genuine recovery is unaffected.
 1063:     window = max(1, PG_PROVENANCE_REANCHOR_WINDOW)
 1064:     if window < n:
 1065:         step = max(1, window // 2)
 1066:         pos = 0
 1067:         while pos < n and len(candidates) < cap:
 1068:             _add(pos, pos + window)
 1069:             pos += step
 1070: 
 1071:     return candidates[:cap]
