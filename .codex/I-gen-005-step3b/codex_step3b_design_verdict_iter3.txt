### Iter-2 novel P1: strict_verify scope under-specified

You correctly noted that strict_verify's internal checks include:
- numeric extraction (covered by iter-2 proposal)
- content-overlap with cited span
- local-window placement check
- entailment judge (default-enforced)

If only numeric extraction strips atom_NNN/ev_XXX/[N], the other checks still see the raw tokens. content-overlap sees "atom_003" and tries to match it in the cited span (fails). entailment judge sees noise.

**Iter-3 fix**: introduce ONE verifier-cleaned sentence string used across all strict_verify internal checks. SentenceVerification.sentence preserves the original (with all citation artifacts). The cleaning function lives in strict_verify or its companion module, applied once at the top of verify_sentence_provenance:

```python
def _verifier_cleaned_text(sentence: str) -> str:
    """Strip citation artifacts before any verifier internal check."""
    s = re.sub(r"\[\d+\]", " ", sentence)              # bibliography markers
    s = re.sub(r"\[?ev_\d+(?::\d+-\d+)?\]?", " ", s)   # ev tokens
    s = re.sub(r"\(?atom_\d{3,}(?:,\s*atom_\d{3,})*\)?", " ", s)  # atom tokens
    return re.sub(r"\s+", " ", s).strip()
```

In `verify_sentence_provenance`:
```python
verifier_text = _verifier_cleaned_text(sentence)
# Numeric extraction uses verifier_text
# content-overlap compares verifier_text vs cited span text
# local-window check positions verifier_text
# entailment judge inputs verifier_text + cited span
# SentenceVerification.sentence = original sentence (preserved)
```

Single point of strip. All internal paths cleaned. Original preserved for downstream (atom validator, citation resolver).
