# Task

POLARIS is a Deep Research (DR) system being delivered to the Canadian PM as a gift in September 2026. We need to audit/evaluate its outputs against the highest current industrial standards. The user has rejected sample-based audits and FAB-only metrics as insufficient; the audit must be (1) line-by-line every claim/citation/reasoning step, (2) applying highest latest industrial standards, (3) parallel-independent Claude+Codex with reconciliation.

**Question:** What is the LATEST (2025-2026) best practice for evaluating Deep Research / long-form retrieval-augmented generation / agentic-research-agent outputs? Specifically:

1. **Academic benchmarks** (2025-2026 papers): BrowseComp, DeepResearchBench, FACTSCORE follow-ons, MiniCheck, OAK, ClaimDecomp, etc.
2. **Industry rubrics** that OpenAI / Anthropic / Google / Perplexity actually use internally or publish for DR systems.
3. **Faithfulness metrics**: RAGAS, TruLens, HHEM 2.1, REFCHECKER, ALCE-WebGPT, FactScore-Bio, MED-FCT, etc.
4. **Clinical/regulatory-specific frameworks** that should be layered on top: PRISMA-AI 2024/2025, AMSTAR-AI, GRADE-AI, MedHELM, OncQA, ClinicianBench, ICMJE AI-COI extensions.
5. **GitHub evaluation harnesses** popular by recent stars/activity (2025-2026): ragas, deepeval, langfuse-eval, opik, etc.
6. **X/Twitter / blog discussions** by credible researchers (Karpathy, Yao Fu, Jason Wei, Sebastian Raschka, Aleksa Gordić, etc.) on current DR eval gaps, multi-claim decomposition, citation-graph audit.
7. **What does the §-1.1 standard (PRISMA 2020 + AMSTAR-2 + GRADE per claim + ICMJE + Cochrane RoB 2 + ROBINS-I + QUADAS-2 + jurisdiction-specific FDA/EMA/HC) miss that the 2026 frontier has added?**
