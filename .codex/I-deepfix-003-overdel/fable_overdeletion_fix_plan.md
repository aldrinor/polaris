## ROOT CAUSE — 3 causes + 2 amplifiers (read from real code + live data)
**Cause 1 (big one): the deletion gate reused a WEIGHT-label predicate as a DELETE trigger.** `junk_deletion_gate.is_row_confirmed_offtopic` (junk_deletion_gate.py:76–92) imports `weighted_enrichment._is_confirmed_offtopic` (weighted_enrichment.py:142–170), which fires on `content_relevance_label in {"demoted","escalated_demoted"}`. Those are NOT topic-judge verdicts — `"demoted"` = Qwen3-Reranker-0.6B numeric score below threshold (content_relevance_judge.py:426, reason literally "demoted, NOT dropped"); `"escalated_demoted"` = GLM "INSUFFICIENT" on entailment (line 764). Both were designed as weight-demote-to-0.25, KEEP-never-drop. The deletion gate imported the predicate unchanged → a noisy reranker score now HARD-DELETES. §-1.3-banned "noisy score as a DROP." 197 deleted rows had topic judge = ON but low reranker score. VM proof: Fed/OECD/ILO/McKinsey/HBS/Wikipedia all show `topic_offtopic_demoted: False` + `content_relevance_label: demoted/escalated_demoted`; none in the resume judge's fresh 349-title demote list — they died on the weight label.
**Cause 2: launcher turned off the positive-relevance safety.** `box1_ideepfix003.sh:56` sets `PG_OFFTOPIC_RELEVANCE_OVERRIDE=0` (an affirmative "relevant" verdict beating a stale off-topic stamp). Copied forward from an earlier cite-surface wave. With it off, 7 rows labeled "relevant"/"escalated_relevant" were deleted.
**Cause 3: failed fetches leak into off-topic (Group A).** `chrome_nonsource=0` yet the deleted set has researchgate "Just a moment…" Cloudflare pages, a garbled WEF PDF, cookie/404 pages — the reranker scored the garbled body low + the judge saw a chrome title → mislabeled `confirmed_offtopic` instead of chrome/recover. The gate isn't separating its two categories.
**Amplifier 1:** the FF4-ASPECT topic prompt (topic_relevance_gate.py:226–259) marks "same entity, different aspect" as OFF — calibrated for demote, not delete. HBR hub died this way.
**Amplifier 2:** stale True stamps from the earlier run in corpus_snapshot; a re-judge batch failed open unparsed (log 7130); fail-open keeps rows without clearing stale stamps → stale stamps fed the delete.
Fail-open for MISSING verdicts DOES work (6 unjudged rows kept). The breach is wrong-signal-deletes, not missing-verdict-deletes.

## FIX PLAN — 6 ordered fixes (each default-ON flag, byte-identical OFF; faithfulness engine untouched; no caps/targets)
1. **Deletion gets its own predicate; weight labels can NEVER delete.** junk_deletion_gate.py new `is_row_deletable_offtopic(row)`: deletes ONLY on the topic judge's affirmative confirmed-OFF stamp; `content_relevance_label` never a trigger; a positive label vetoes deletion unconditionally. Flag `PG_DELETE_OFFTOPIC_TOPIC_JUDGE_ONLY`. Stops the 197-row leg.
2. **Fresh-verdict-only deletion; stale stamps demote-keep.** run_honest_sweep_r3.py deletion seam (14871–14918) + gate signature: only ids THIS run's judge confirmed OFF (`TopicGateResult.demoted_rows`) are deletable; stale snapshot stamp → sibling demote-keep. One decision surface. Flag `PG_DELETE_OFFTOPIC_FRESH_VERDICT_ONLY`.
3. **Split OFF → OFF_SUBJECT (deletable) vs OFF_ASPECT (demote-keep).** topic_relevance_gate.py prompt returns `ON | OFF_ASPECT | OFF_SUBJECT`. OFF_SUBJECT = clearly different subject (scholar-mill/space/religion junk). OFF_ASPECT = same entity wrong aspect (education-AI, HBR hub) → demote never delete. Legacy "OFF" ⇒ OFF_ASPECT (conservative). Sidecar `topic_off_subject=True`; deletion keys on it. Flag `PG_TOPIC_GATE_SUBJECT_ASPECT_SPLIT`.
4. **Route failed fetches to the chrome path, never off-topic.** run_honest_sweep_r3.py content-integrity stamp (~14820–14864) + topic gate batch builder: (a) chrome/garbled/empty body → A15-refetch → Zyte → chrome-delete reason `content_integrity_junk:<class>`, excluded from off-topic; (b) topic judge never judges a chrome row. Makes `chrome_nonsource` honest. Flag `PG_JUNK_CHROME_BEFORE_OFFTOPIC`. Dovetails with I-deepfix-004 front-matter recover-degrade (which owns the scholar-mill deletions).
5. **Disclosure durable + per-source + everywhere.** junk_deletion_gate.disclosure_records + run_honest_sweep_r3.py: (a) per-source reason (signal fired / judge verdict / title / url / tier); (b) write `junk_deletion_disclosure.json` into run_dir IMMEDIATELY at the seam (durable on abort); (c) attach `deleted_*` on EVERY manifest write (abort sites 9195/9622/10507/12025/12344/15917 + unified-status ~18023 + success 17878–17890) via `_attach_junk_deletion_disclosure(manifest)`; (d) the promised Methods line in report.md.
6. **Launcher hygiene.** Remove `PG_OFFTOPIC_RELEVANCE_OVERRIDE=0` from box1 launch scripts.
