## Findings
### P0
- The plan’s signed-bundle premise is not honest enough to approve. [docs/stier_experience_plan.md](C:/POLARIS/docs/stier_experience_plan.md:56) and [docs/stier_experience_plan.md](C:/POLARIS/docs/stier_experience_plan.md:138) say the hero uses an “existing real signed bundle,” and §14 promises offline receipt handoff. But the live screenshot shows “Signature missing,” and local `web/public/canonical_bundles/v1_canonical_success` has no `manifest.yaml.asc`; the loader only marks `signaturePresent` when that file exists and is non-empty ([web/lib/inspector_bundle_loader.ts](C:/POLARIS/web/lib/inspector_bundle_loader.ts:142)). Test fixtures explicitly say their `.asc` is a placeholder, not a real key signature ([tests/fixtures/signed_bundle/README.md](C:/POLARIS/tests/fixtures/signed_bundle/README.md:15)). Fix: add a hard pre-step that produces and dry-run-verifies a real signed PM demo bundle, blocks hero/demo signoff on missing/placeholder signatures, and forbids “signed” language unless GPG verification succeeds.

### P1
- Abstention and SoF gates are still named, not operationalized. §5 says “below a defined adequacy/certainty floor,” but the plan should specify the launch gate: required outcomes for clinical demo, minimum source/body adequacy, how multi-outcome claims map to SoF certainty, and when the UI must refuse instead of badge. This is no longer a GRADE-method P0, but it is execution-risky.
- Verifier reliability needs a release gate, not only a display promise. §7 says “measured reliability,” but the plan should require the actual gold-set metric threshold, calibration examples, and disagreement escalation before the PM build can call the independent-family check trustworthy.

### P2 / P3
- In §6, add explicit “not for individual-patient recommendations” and “not for urgent/time-sensitive clinical decisions.” That aligns better with Health Canada/FDA non-device CDS framing.
- Current live Home/Intake/Inspector remain not gift-grade, but §13 honestly calls them structural rebuilds, so this is not a plan blocker.
