## 1. Goal

Build `src/polaris_graph/authority/credibility_skill.py`: one generic, domain-agnostic LLM "credibility skill" that, per research question, scores EACH candidate source on **reliability × relevance** with a written, inspectable rationale, consuming POLARIS's already-computed deterministic authority signals as its priors. This is the Layer-1 scorer of the credibility-weighted sourcing redesign (`docs/credibility_weighted_sourcing_redesign_plan_2026_06_07.md` §9.1, research `docs/credibility_adaptive_weighting_and_bothsides_recs_2026_06_08.md`).

## 2. Mechanism (JudgeRank-style 3-step, each step inspectable = the audit template)

The injected judge is asked, for the question + one source at a time (or a small batch), to produce:
1. **query_need** — what THIS question needs; what would count as strong evidence for it (string).
2. **source_assessment** — query-aware, applying GENERIC SIFT + lateral-reading + provenance heuristics (prefer primary/official over aggregator/SEO; original over secondary summary; corroborate laterally; weigh incentives/track record; downgrade stale/superseded/retracted). NOT a domain table.
3. **two-axis judgment** — `reliability_score ∈ [0,1]` (internal validity/method given the signals) × `relevance_score ∈ [0,1]` (directness to this question), a `credibility_weight` (their product, clamped), a `rationale` string, and `signals_cited` (the deterministic signal names it reasoned from).

The skill's PRIORS are the deterministic `authority_model` signals. It may up/down-weight them only with a stated reason; it MAY NOT fabricate authority where `authority_confidence == "LOW"` (a thin-signal source stays honestly low — this is an explicit guardrail in the prompt AND asserted by a test with a deterministic fake judge that tries to over-claim).

- `judge=None` ⇒ NO network: return a priors-only judgment per row (reliability seeded from `authority_score`, relevance unknown→neutral, rationale="no judge wired", `signals_cited` from present signals). This keeps the function pure + offline-testable AND makes off-mode a no-op.
- When a judge IS injected: per-source call; a judge error sets `judge_error=True` for THAT source and falls back to the priors-only judgment for it (fail-loud via the flag, never silently fabricate). The deterministic priors are the floor.
- The judge callable is wired by the CALLER (a future phase), exactly like `get_default_judge()` in `semantic_conflict_detector.py`; this pure library constructs no client. Model/endpoint/key come from env (`OPENROUTER_API_KEY`, a `PG_CREDIBILITY_SKILL_MODEL` knob), never hardcoded.
- The judge's returned scores are CLAMPED to [0,1] and its `signals_cited` are intersected with the signals actually present on the row (it cannot cite a signal that wasn't given — anti-hallucination).

## 4. Acceptance criteria (each maps to a test, offline, deterministic fake judge — no network, no live data)

2. `judge=None` ⇒ priors-only: returns one judgment per row, no exception, reliability seeded from `authority_score`, `judge_error=False`, no network (assert no httpx import path hit — structurally guaranteed since no client is built).
3. Injected fake judge ⇒ its reliability/relevance/rationale flow through; `credibility_weight == clamp(reliability*relevance)`.
4. Anti-fabrication guardrail: a fake judge that returns reliability=0.99 for a row whose `authority_confidence == "LOW"` and whose signals are thin is CLAMPED/floored per the guardrail (the high score does not survive unchecked) — assert the documented behaviour.
5. `signals_cited` is a SUBSET of the signals present on the row (a cited-but-absent signal is dropped).
6. Out-of-range judge scores (`reliability=1.7`, `-0.2`) are clamped to [0,1].
7. A judge that raises for one row sets `judge_error=True` for THAT row only and falls back to priors; other rows are unaffected (isolation, recall-first).
8. Domain is a HINT: passing `domain="clinical"` vs `domain="policy"` does NOT change the code path / dataclass shape (same fields), proving no fixed-rubric branch.
9. Score-combiner thresholds are env-overridable (a `PG_CREDIBILITY_*` knob changes the combiner) — LAW VI.
10. Purity: no faithfulness file imported; `score_source_credibility` mutates no input row (assert input rows unchanged by identity/content).

- CONTRADICT: the two claims cannot both be true for the same population/endpoint — one asserts something the other explicitly denies or reverses (e.g. "improved overall survival" vs "no overall survival benefit"; "first-line therapy" vs "reserved for refractory cases").
