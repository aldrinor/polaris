### A1. Best practice — COMPOSITION (how winners compose one coherent report)

How the best 2026 systems compose one coherent report (grounded in packs 1-2). The winning ordering is COMPOSE-THEN-VERIFY, not verify-then-assemble-atoms. Concrete pipeline: (1) GLOBAL PLAN / THESIS FIRST. Evidence: GPT-5 Deep Research scores 87.5 on Relevant-Statement vs list-dumping agents at 12-22, which proves winners write coherent multi-paragraph synthesis with a through-line, not a source-list tail. STORM (named in pack 1 reference designs) is outline-driven multi-perspective synthesis; its perspective-guided question-asking surfaces pro and con BEFORE writing, yielding both an outline and two-sidedness. DRB-II Analysis rubrics demand conclusions logically grounded in evidence, i.e. an argument, not aggregation. (2) SYNTHESIZE-THEN-VERIFY (the core pattern). LongCite CoF: generate the coherent long-form answer FIRST as ordinary prose, THEN attach fine-grained sentence-span citations coarse-to-fine (chunk-retrieve, expand neighbors, number sentences, extract [a-b] spans). Pack 1 states it directly for POLARIS: compose the coherent basket-paragraph and cross-basket thesis first as prose, then run strict_verify/NLI to snap each sentence to its supporting span(s), dropping or hedging failures. This is the opposite of composing at atom granularity. (3) NO ISLANDS / COHERENCE EDIT. DeepTRACE Relevant-Statement scores filler, transitions, off-topic prose and list-fragments as 0. The move: delete the additive source-by-source list tail and replace it with basket-PARAGRAPHS whose every sentence stays span-certified, plus a cross-basket thesis that connects baskets so nothing is orphaned. C-GRPO connectivity (a rubric counts only if BFS-connected to the final answer through shared entities) is the training-time analog of no-islands. (4) TWO-SIDED SYNTHESIS for debate queries: place at least one supported pro AND one supported con per debate claim with hedged certainty (DeepTRACE One-Sided / Overconfident). Pure per-atom grounding cannot produce this; it needs the synthesis layer. So the composition recipe is: STORM outline/thesis with pro-con perspectives, then draft fluent basket-paragraphs + cross-basket thesis (LongCite compose-first), then a coherence edit that kills filler and islands, then hand off to the attribute-verify-repair loop (next field).

### A2. Best practice — COVERAGE + ANALYSIS (winning DRB-II analysis/insight)

Concrete methods that win DRB-II Analysis/insight, not just recall (grounded in pack 2, plus C-GRPO from pack 1). (1) SEPARATE ANALYSIS FROM RECALL. DRB-II scores three dimensions independently: InfoRecall (retrieve the right facts), Analysis (synthesis beyond aggregation), Presentation (clear, verifiable, structured). Analysis rubrics explicitly encode reasoning or synthesis rather than direct factual lookup and require conclusions logically grounded in the evidence. So to win Analysis you must emit higher-order INFERENCE claims (why, implication, cross-comparison, trend, causal link) that are each still span-grounded, not just content-bearing lookups. (2) THE RUBRIC SHAPE tells you exactly what a high-coverage report must contain: Essential (necessary for a correct answer), Atomic (one fact or one inference; split compound points), Content-bearing (encodes the actual answer, e.g. X increased from A to B between year Y and Z, NOT discusses X), Numerically-precise (explicit values with a tolerated error range). POLARIS must therefore emit content-bearing numerically-precise atomic recall claims AND separate atomic analysis/inference claims. (3) BLOCKED-REFERENCE FORCES INDEPENDENT MULTI-SOURCE RECALL. The seed expert article and its references are blocked; scoring is ternary: 1 if the rubric is satisfied AND supporting sentences do NOT cite a blocked ref, 0 if not mentioned, -1 if mentioned but supported by a blocked ref. This rewards genuine breadth from OTHER sources, which aligns perfectly with POLARIS weight-and-consolidate DNA (many independent sources per claim). Coverage that wins = broad honest independent retrieval so the fact is available from a non-blocked source. (4) RUBRIC SKEW: per task avg InfoRecall 52.9, Analysis 12.77, Presentation 5.65, so TotalScore is de-facto recall-dominated; but recall saturates near the top, so the MARGINAL win is Analysis synthesis plus structured Presentation (an example Presentation rubric is: presented the change in gold prices in table form, so emitting tables is directly scored). (5) C-GRPO is the coverage-OPTIMIZATION method: reward = fraction of answer-rubrics that are identified AND citation-supported AND connected to the thesis, applied ONLY when the outcome answer is correct, URL-capped at 20 to stop citation-count hacking. It ties coverage to grounding to coherence in one objective, so it lifts recall/analysis without ungrounded padding. (6) DRB-I Insight/Depth uses dynamic task-adaptive criteria scored against a human reference, so depth is measured as task-specific insight criteria satisfied, not word count.

### A3. Best practice — TRACEABILITY (draft->attribute->verify->repair, wins DeepTRACE)

Concrete draft to attribute to verify to repair loop that wins DeepTRACE while staying fluent (grounded in pack 1). (1) DRAFT: compose fluent prose first (LongCite compose-first) so the text reads as connected paragraphs, keeping Relevant-Statement high. (2) ATTRIBUTE coarse-to-fine (LongCite CoF): per answer sentence retrieve 128-token chunks, expand each cited chunk with neighbors, number the complete sentences, extract fine-grained [a-b] sentence-span snippets. Attach the MINIMAL per-sentence supporting set where each cited source independently entails that sentence. (3) VERIFY per-sentence with NLI (ALCE): recall(s_i)=1 iff C_i non-empty AND phi(concat(C_i),s_i)=1; a citation c_ij is irrelevant iff phi(c_ij,s_i)=0 AND phi(concat(C_i minus c_ij),s_i)=1 (leave-one-out necessity); use TRUE/T5-11B (or equivalent) as phi. DeepTRACE F-matrix is the matrix generalization: F[i,j]=1 iff source j fully/partially supports statement i, judged by reading the source FULL text and IGNORING citation markup. (4) REPAIR against the metric mechanics, never relaxing the gate: drop or hedge any sentence that fails entailment (fixes Unsupported, which GPT-5 DR holds at 12.5 vs others 50-97); PRUNE over-citations where a cited source does not itself entail the sentence, because Citation-Accuracy = sum(C AND F)/sum(C) turns 5-cited-but-3-support into 60 percent; ADD missing true supporters because Citation-Thoroughness = sum(C AND F)/sum(F) rewards citing every real supporter; DEMOTE redundant sources that cover the same statement-set, because Source-Necessity via min-vertex-cover penalizes redundancy; REMOVE listed-but-uncited sources (Uncited-Sources). The fluency-vs-grounding reconciliation: because you compose fluent first and verify second and repair only by drop/hedge/re-attribute, prose stays fluent (high Relevant-Statement) while every surviving sentence is span-entailed (low Unsupported). This is exactly the failure Liu-2023 measured (only 51.5 of sentences fully supported, 74.5 of citations support) turned into a fix. (5) SELF-PREDICT OFFLINE: build an ALCE-style predictor (TRUE-NLI recall + leave-one-out precision over C AND F) to estimate DeepTRACE Accuracy/Thoroughness/Unsupported BEFORE any paid GPT-5 judge run, matching the operator prove-our-own-scorer-then-trust-it rule. (6) TRAINING-TIME ALTERNATIVE to post-hoc repair: C-GRPO bakes citation-support into the RL reward so grounding is optimized jointly with coverage.

### A4. THE reference architecture (end-to-end, coherence-vs-per-sentence-verify reconciliation)

The exact end-to-end pipeline the winners use, with the coherence-vs-per-sentence-verification reconciliation stated crisply. THE RECONCILIATION (the key insight): coherence is produced at COMPOSE time over whole baskets and the cross-basket thesis; per-sentence verification runs DOWNSTREAM as a GATE that can drop, hedge, or re-attribute a sentence but never re-fragments the prose and is never the compositor. **Concrete methods:** DeepTRACE is a post-hoc audit, not a generator. Pipeline: (1) STATEMENT DECOMPOSITION — a GPT-5 extraction prompt splits the report into individual sentences, preserving inline citation markup like "[1]", and labels each sentence core=1 (carries query-relevant info) or filler=0 (intro/conclusion/transition). Sentences bearing a citation are "almost always core." (2) Build two binary statements x sources matrices: CITATION MATRIX C (C[i,j]=1 iff statement i's markup cites source j, parsed mechanically from text) and FACTUAL-SUPPORT MATRIX F (F[i,j]=1 iff source j actually supports statement i, decided by the GPT-5 judge reading the source's fetched full text against the statement, explicitly IGNORING the citation markup). (3) CONFIDENCE — a separate GPT-5 pass rates the whole answer's language certainty on a 1-5 Likert. (4) SOURCE NECESSITY — F is read as a (statement,source) bipartite graph; the minimal set of sources that still covers every supported statement is found via Hopcroft-Karp minimum-vertex-cover; necessity = |MVC|/|listed sources|. (5) The 8 metrics are ratios/binaries computed from C, F, the core/filler labels, and confidence. ~15% of URLs fail extraction (paywall/404) and are dropped. The whole thing is ~80,000 judge calls per system sweep.

ALCE lineage (the scoring engine DeepTRACE's citation matrices inherit): citation quality is decided by an NLI entailment model phi (TRUE, a T5-11B fine-tuned on SNLI+MNLI+FEVER+SciTail+PAWS+VitaminC). Recall for sentence s_i = 1 iff its citation set C_i is non-empty AND phi(concat(C_i), s_i)=1 (the concatenation of all cited passages entails the sentence). Precision marks a single citation c_ij "irrelevant" iff BOTH phi(c_ij, s_i)=0 (it alone does not entail) AND phi(concat(C_i minus c_ij), s_i)=1 (removing it does not break entailment) — i.e. a leave-one-out necessity test; c_ij scores precision=1 only if s_i has recall=1 and c_ij is not irrelevant. Max 3 citations per statement. DeepTRACE's F-matrix + Citation-Accuracy/Thoroughness is the matrix generalization of exactly this: F[i,j] is the per-source analog of ALCE's phi() support call.

Current SOTA method (2601.06021, C-GRPO): rather than post-hoc filtering, it trains the agent with a citation-aware rubric reward so grounding and coverage are optimized jointly. R_rubric = |R_connect| / |R_total| = fraction of answer-rubrics that are (a) identified, (b) citation-supported against fetched web content, AND (c) connected to the final answer via a bipartite-graph BFS over hidden entities. Mixed reward R_i = (1-alpha)*R_outcome + alpha*R_outcome*R_rubric with alpha=0.3, and rubric reward only applied when outcome reward=1 (so it cannot reward well-cited wrong answers). Caps at 20 cited URLs to stop citation-count reward-hacking.

**Exact details (formulas / steps / prompts / algorithms):** THE 8 DEEPTRACE METRICS (verbatim formulas; C=citation matrix, F=factual-support matrix, ⊙=element-wise product, Σ=sum over all cells):

1. ONE-SIDED ANSWER (binary, debate queries only) = 0 if BOTH a pro AND a con statement are present, else 1. LOWER better. Thresholds: good [0,20), borderline [20,40), problematic [40,100).
2. OVERCONFIDENT ANSWER (binary, debate queries) = 1 if (One-Sided=1 AND whole-answer Confidence=5), else 0. LOWER better. Same thresholds.
3. RELEVANT STATEMENT (ratio) = (# relevant/core statements) / (total statements). HIGHER better. good [90,100), borderline [70,90), problematic [0,70).
4. UNCITED SOURCES (ratio) = (# cited sources) / (# listed sources). Reported as %uncited elsewhere; higher cited-fraction better. good [0,5) uncited, problematic [10,100).
5. UNSUPPORTED STATEMENTS (ratio) = (# unsupported statements) / (# relevant statements). A relevant statement is unsupported if no listed source supports it in F. LOWER better. good [0,10), borderline [10,25), problematic [25,100).
6. SOURCE NECESSITY (ratio) = (# necessary sources = |min-vertex-cover|) / (# listed sources), via Hopcroft-Karp on the F bipartite graph. HIGHER better. good [80,100), borderline [60,80), problematic [0,60).
7. CITATION ACCURACY (= precision) = Σ(C ⊙ F) / Σ(C). Of all citations the report makes, the fraction that land on a source that genuinely supports that statement. HIGHER better. good [90,100), borderline [50,90), problematic [0,50).
8. CITATION THOROUGHNESS (= recall) = Σ(C ⊙ F) / Σ(F). Of all true (statement,source) support pairs that exist, the fraction the report actually cited. HIGHER better. good [50,100), borderline [20,50), problematic [0,20).

LONGCITE CoF (4 steps): (1) LLM self-generates query+answer from long context; (2) split context into 128-token chunks, retrieve chunks per answer sentence, LLM segments answer into statements + emits chunk-level citations; (3) expand each cited chunk with neighbors, number the complete sentences, LLM extracts fine-grained sentence-span snippets as number spans [a-b]; (4) discard instance if <20% of statements have citations. LongBench-Cite: recall 0/0.5/1 (GPT-4o: fully/partially/not supports); precision binary (a citation is relevant iff it entails some key point of the statement); F1=2PR/(P+R); citation length = avg tokens of cited snippets (lower=finer). LongCite-45k = 44,600 filtered instances from 50k GLM-4 corpus docs, 256-128k tokens, 9 domains.

C-GRPO (2026): R_rubric=|R_connect|/|R_total|; R_i=(1-alpha)R_out + alpha*R_out*R_rubric, alpha=0.3, rubric only when outcome=1; ≤20 cited URLs; judge accuracy 97.7% entity-ID / 95.1% citation-rubric.

**How it scores high:** Metric-by-metric, exactly what a report must do to win DeepTRACE, and the mechanism:

- CITATION ACCURACY (Σ(C⊙F)/Σ(C)): every source you cite on a sentence must be one the GPT-5 judge, reading that source's FULL fetched text, rules "full/partial support" for THAT sentence. The killer: over-citation. If you attach 5 sources to a sentence but only 3 genuinely support it, accuracy = 3/5 = 60%. Span-level provenance (POLARIS strict_verify) maxes this IF and only if each cited span independently supports the sentence.
- CITATION THOROUGHNESS (Σ(C⊙F)/Σ(F)): you must cite EVERY source that in fact supports each sentence, not just one. Under-citing corroboration hurts recall. This is the metric that REWARDS multi-citation baskets.
- UNSUPPORTED STATEMENTS: every core sentence needs at least one supporting source in F. Grounding-first composition (never emit a sentence without a verified span) drives this toward 0 — GPT-5(DR) hits 12.5%, everyone else 50-97%. This is POLARIS's natural strength.
- RELEVANT STATEMENT ratio: filler/chrome/transition/off-topic prose is scored 0 and tanks the ratio. GPT-5(DR)=87.5% vs Gemini(DR)=12.4%. This is where POLARIS's choppy atom prose + additive source-list tail BLEEDS: disconnected list-like fragments and boilerplate read as filler.
- ONE-SIDED / OVERCONFIDENT: only debate queries. You must synthesize BOTH a pro and a con statement in the same answer; otherwise one-sided=1, and if you also sound maximally certain, overconfident=1. Even GPT-5(DR) stays one-sided (54.67%) — this is the single hardest metric and requires deliberate cross-source pro/con synthesis, plus hedged language. Pure per-atom grounding CANNOT produce this; it needs a synthesis layer.
- SOURCE NECESSITY (|MVC|/|listed|): if two listed sources support the exact same set of statements, only one counts as necessary — redundancy is penalized. Listing sources you never cite (or that duplicate coverage) drops this below the 60% floor.
- UNCITED SOURCES: never list a source in the bibliography that no sentence cites (creates false-validation). Listed sources must equal cited sources.

The winning profile (GPT-5 Deep Research): near-zero unsupported, ~88% thoroughness, ~79% accuracy, ~88% relevant, ~88% source necessity — achieved by grounding every sentence AND writing tight coherent prose with no filler AND citing all true supporters — while STILL failing one-sidedness. So "high faithfulness + depth" = grounding-first + no-filler coherent synthesis + explicit two-sided balance, and the accuracy/necessity metrics quietly PENALIZE naive keep-all-sources over-citation.

**Reference system designs:** The actual DeepTRACE-topping system is OpenAI GPT-5 Deep Research (GPT-5(DR)): it is the only tested system that is simultaneously near-best on relevant-statements (87.5), unsupported (12.5, best), source-necessity (87.5), accuracy (79.1) and thoroughness (87.5). Its design behavior, inferred from the metric profile: it writes coherent multi-paragraph synthesis (high relevant-statement, unlike the list-dumping agents at 12-45%), grounds essentially every sentence (low unsupported), and cites the full supporting set per claim (high thoroughness) without listing dead sources (0% uncited). Its remaining weakness is debate-balance (one-sided 54.67%). The other DR agents (Perplexity, Gemini, Copilot) collapse on relevant-statement (12-22%) and unsupported (54-97%) — they retrieve broadly but dump loosely-attributed source lists, exactly POLARIS's current failure mode.

The best METHOD design for this area is now the 2026 citation-aware-RL family (C-GRPO / DeepDive): instead of post-hoc filtering, train the generator so the reward equals coverage-rubrics-that-are-citation-supported-AND-connected-to-the-thesis (bipartite BFS over entities), applied only on correct answers, URL-capped to stop over-citation hacking. This is the one design that provably ties depth (rubric coverage) to grounding (citation support) to coherence (connectivity to the final answer) in a single objective.

The composition-depth reference design is LongCite's CoF: generate the coherent long-form answer FIRST, THEN attach fine-grained sentence-span citations coarse-to-fine (chunk-level retrieve → expand → number sentences → extract [a-b] spans). This is the concrete recipe for "coherent prose with clean per-sentence grounding" — the opposite of composing at atom granularity. STORM (outline-driven multi-perspective synthesis) is the reference for the two-sided/one-sidedness axis: its perspective-guided question asking naturally surfaces pro and con before writing.

ALCE (TRUE T5-11B NLI + leave-one-out precision) is the reference scoring engine POLARIS's own faithfulness harness should mirror to self-predict DeepTRACE accuracy/thoroughness offline before any paid judge run.

**Applicability to POLARIS:** POLARIS's confirmed root cause maps precisely onto specific DeepTRACE metrics, and the fix must respect the never-relax-faithfulness constraint:

**Concrete methods:** TWO SEPARATE BENCHMARKS, TWO METHODOLOGIES.

DRB-II (arXiv:2601.08536) — RUBRIC-BASED, absolute. 132 tasks (66 EN, 66 ZH), 22 domains, 9,430 binary rubrics. Each task is seeded from ONE open-access expert investigative/review article (the gold source). Rubrics are decomposed from that article across 3 dimensions: InfoRecall (retrieve the right facts), Analysis (synthesize higher-level insight beyond aggregation), Presentation (clear/verifiable communication). Rubric construction = 4-stage pipeline: (1) LLM-guided extraction from the expert article with designed prompts; (2) SELF-EVALUATION ITERATION — apply each freshly-generated rubric back onto its own source article via the eval script; if InfoRecall or Analysis accuracy on the source is <90%, treat the rubric as hallucinated/unreliable and re-prompt the LLM; (3) manual human revision (redundancy removal, atomicity, wording); (4) expert review (>400 human-hours). Judge = Gemini-2.5-Pro, binary per rubric, run in batches of 50. THE KEY MECHANISM: the seed article and its refs are 'BLOCKED' — the agent must independently retrieve the same facts from OTHER web sources; scoring is TERNARY not binary at the evidence level: score=1 (rubric satisfied AND supporting sentences do NOT cite a blocked ref), score=0 (not mentioned at all), score=-1 (mentioned BUT the supporting sentences cite a blocked reference). This is what forces genuine independent multi-source recall rather than parroting the gold article.

DRB-I (arXiv:2506.11763) — REFERENCE-BASED RELATIVE (RACE) + citation audit (FACT). 100 tasks, 22 domains. RACE = Reference-based Adaptive Criteria-driven Evaluation with dynamic weighting; 4 dimensions: Comprehensiveness, Insight/Depth, Instruction-Following, Readability. For each task an LLM DYNAMICALLY GENERATES task-specific criteria under each dimension (each with a weight; criteria weights sum to 1.0 within a dimension, and dimension weights also sum to 1.0, both task-adaptive). A judge scores BOTH the target report and a human expert REFERENCE report 0-10 on every criterion, then normalizes the target against the reference. FACT = separate pipeline: extract (statement, URL) pairs from the report → dedup → scrape the URL + LLM-judge whether the source actually supports the statement → citation accuracy + effective-citation count.

**Exact details (formulas / steps / prompts / algorithms):** DRB-II SCORING. Final task score = (number of rubrics scored 1) / (total rubrics). Per-dimension score = same fraction restricted to that dimension's rubrics. NO dimension weighting — but because rubric COUNTS are lopsided (avg InfoRecall 52.90, Analysis 12.77, Presentation 5.65 per task; ~71.3 total), the TotalScore is de-facto dominated by InfoRecall. Judge output JSON per rubric item: {\"rubric_item\": <MUST be character-exact copy of input>, \"score\": 1|0|-1, \"reason\": <brief>, \"evidence\": <supporting sentences or \"\">}. Judge INPUT serialization: {\"task\": ..., \"rubric_items\": [...], \"blocked\": {\"title\":..., \"authors\":[...], \"urls\":[...]}}. Rubric DESIGN PRINCIPLES (from README): Essential (necessary for a correct answer), Atomic (single fact/inference; split complex points), Content-bearing (encodes the actual answer, e.g. 'X increased from A to B between years Y and Z' — NOT 'discusses X'), Numerically-precise (explicit values with tolerated error range). Analysis rubrics must 'encode reasoning or synthesis rather than direct factual lookup' and require 'conclusions logically grounded in the evidence.' Example Presentation rubric: 'Presented the change in gold prices in table form.' Aggregation script emits agg_scores_{inforecall,analysis,presentation,total,blocked}.csv.

DRB-I RACE FORMULA (exact): S_final(R_tgt) = S_int(R_tgt) / ( S_int(R_tgt) + S_int(R_ref) ), where S_int is the criteria-weighted, dimension-weighted intrinsic 0-10 score. A report exactly as good as the reference scores 0.5; the published leaderboard numbers (~44-49) are these values ×100. S_int = Σ_dims [ dim_weight × Σ_criteria (criterion_weight × judge_score_0to10) ]. The 0-10 scale rubric given to the judge: 0-2 very poor / 2-4 poor / 4-6 average / 6-8 good / 8-10 excellent. Judge scores <article_1>=target and <article_2>=reference on identical criteria. ABLATION finding: removing reference-based comparison caused the LARGEST drop in human-agreement — the reference normalization is the single most load-bearing component. Human inter-annotator agreement baseline = 68.78%.

DRB-I INSIGHT/DEPTH definition (verbatim intent from criteria_prompt_en.py): 'The depth, originality, logic, and value of the analysis and conclusions' — assess 'analytical depth, logical rigor, originality of insights, and value of conclusions, not just information listing.' HIGH insight = identifies 'key drivers of change', establishes 'sophisticated analytical frameworks', offers 'forward-looking, actionable recommendations' 'beyond simple data presentation.' Comprehensiveness = 'breadth, depth, and relevance of information coverage.'

FACT: citation_accuracy = (# statement-URL pairs where scraped source supports the statement) / (# total extracted statement-URL pairs); effective_citations = average count of verifiably-supported citations per task (e.g. Gemini-2.5-Pro DR = 111.21; OpenAI DR = 40.79; Perplexity = 31.26).

**How it scores high:** TO WIN DRB-II ANALYSIS (the dimension POLARIS's islands lose): Analysis rubrics reward CROSS-SOURCE SYNTHESIS statements — a claim like 'the primary driver of X was Y, which explains the divergence between A and B' that is (a) grounded in evidence and (b) NOT a lookup. A source-by-source list scores 0 on nearly every Analysis rubric because no single-source atom asserts the higher-order relation. You win by producing analytical sentences that assert relations/causes/comparisons/implications spanning multiple baskets, each still evidence-grounded. Crucially the -1/blocked mechanism means: to satisfy an InfoRecall rubric you must corroborate the fact from a NON-gold source; a report that cites only the seed article scores -1. So breadth of independent corroboration (POLARIS's basket multi-attribution) directly converts to InfoRecall points, and multi-basket synthesis converts to Analysis points. Presentation rubrics are literal format checks (tables, sections) — cheap, high-value (avg only 5.65 items but easy to satisfy; note top agents already hit 83-92 here).

TO WIN DRB-I: because the score is target/(target+ref) vs a HUMAN EXPERT reference, you only need to match-or-beat the expert per criterion. Insight/Depth is won by an explicit analytical FRAMEWORK + a stated global thesis + forward-looking recommendations, exactly the things a choppy island report lacks. Comprehensiveness rewards breadth×depth×relevance. FACT is won by high statement-level citation support (every claim's cited URL must actually support it on scrape) AND high volume of supported citations — POLARIS's strict_verify/provenance already targets the accuracy half; effective-citation COUNT rewards breadth.

The two benchmarks pull in the SAME direction for POLARIS: independent multi-source corroboration (recall/FACT) + genuine cross-source synthesis with a thesis (Analysis/Insight) + clean formatting (Presentation/Readability).

**Concrete methods:** FIVE named, reproducible methods, all converging on SEPARATE-PLAN-FROM-WRITE + COMPOSE-AT-SECTION-GRANULARITY-NOT-ATOM.

1) WebWeaver (the single most on-point). A two-agent ReAct system. The PLANNER runs a dynamic cycle with three parsable actions {search, write_outline, terminate}. It does NOT emit a one-shot outline: it interleaves evidence acquisition with continuous outline restructuring, and every outline section is annotated with the citation IDs of the specific evidence that supports it (<citation>id_2, id_5, id_7, id_18, id_22</citation>). Evidence is not held in context — it is written to a MEMORY BANK keyed by citation ID. The WRITER then executes a separate hierarchical loop {retrieve, write, terminate}: for section k it retrieves ONLY that section's cited evidence IDs from the memory bank, reasons over them + prior sections, writes the section as one narrative, then PRUNES that section's source material out of context (replaced by a placeholder) before the next section. This "dynamic retrieve-and-prune" is what kills both long-context degradation and cross-section interference (the islands problem).

2) TTD-DR (Google; the 'global thesis first, then fill' answer). Models report writing as a DIFFUSION process. It first drafts a preliminary full report ('noisy draft') mostly from the model's parametric knowledge — an updatable SKELETON that is the evolving foundation for the whole report. It then 'denoises' the draft over up to N=20 steps: at each step it generates the next search question CONDITIONED ON the current draft, retrieves a synthesized answer, and revises the WHOLE draft with the new info. Because there is always one global draft, global context/coherence is never lost. Layered on top is component-wise SELF-EVOLUTION: for each stage (plan/question/answer/report) it spawns diverse variants, scores each with an LLM-as-judge on Helpfulness + Comprehensiveness, revises toward higher fitness, then CROSS-OVER merges the best variants into one output.

3) STORM (the foundational plan-then-write baseline). Two stages. PRE-WRITING: discover N=5 perspectives (prompt LLM for related topics → pull those Wikipedia articles' tables-of-contents → prompt for N complementary perspectives, plus a fixed 'basic fact writer' baseline perspective p0), then for each perspective simulate a multi-turn conversation (max M=5 rounds) between a writer-with-perspective and a retrieval-grounded topic expert; every answer is grounded on filtered sources added to reference set R. WRITING: generate a draft outline from topic alone, REFINE it using all the simulated conversations, then write section-by-section (retrieve from R by Sentence-BERT similarity to section title), then a final dedup + lead-section pass.

4) Co-STORM (the 'N sources → ONE structure, not a list' answer). Three roles — user, N expert agents, and a MODERATOR — take turns via a discourse protocol. The consolidation engine is a dynamic hierarchical MIND MAP M=(C,E): a concept tree where directed edges are parent-child topic relations. Every retrieved snippet is INSERTED under the most appropriate concept (semantic-similarity shortlist of candidate concepts, then LLM picks final placement); when a concept overflows, REORGANIZE prompts the LLM to invent new subtopics and re-insert; a bottom-up CLEANUP deletes empty concepts and collapses single-child concepts. The final report uses the mind map itself as the outline (tree traversal) and writes each section from that concept's accumulated info.

**Concrete methods:** THE ARCHITECTURE ROOT-CAUSE FIX (2604.01432): citation granularity is a chunk size k = number of consecutive sentences the source is pre-segmented into; k=1 is POLARIS's current atom mode. Attribution F1 is NOT monotone in fineness — it PEAKS at intermediate k (paragraph scale, k=4-8) and collapses at k=1. The remedy is not tuning: it is raising the certified unit to a paragraph and certifying it against a BASKET (union) of spans. Two failure modes at k=1 they name: (a) boundary/localization noise — the model finds the right region but not exact syntactic start/end, coarser chunks absorb this; (b) coreference/multi-sentence-reasoning breakage — k=1 isolates a subject from its relation across sentences, so the atom cannot be independently supported even when the answer is correct.

THE SYNTHESIS-COHERENCE FIX (WebWeaver) — the direct answer to POLARIS's 'islands with no global thesis': a two-agent loop. (1) PLANNER runs a dynamic agentic cycle — Search (query web, LLM-filter URLs, summarize, extract evidence -> store in a MEMORY BANK with unique IDs) interleaved with Write-Outline (build/refine a citation-grounded outline where each section/subsection node carries <citation>id_5, id_12</citation> links to memory), iterating until Terminate. The outline IS the global thesis, produced BEFORE writing, co-evolved with evidence. (2) WRITER composes section-by-section (ReAct): for the current section it retrieves ONLY that section's cited evidence from memory, reasons over prior sections + that subset, writes prose in <write> tags, then PRUNES the finished section's raw evidence from context (replaced by a placeholder). This keeps per-step input small (deep synthesis, no lost-in-the-middle) while carrying prior prose forward for global narrative coherence.

THE DRAFT->ATTRIBUTE->VERIFY->REPAIR LOOP (RARR + CoVe): RARR is post-hoc attribution+repair that PRESERVES prose instead of dropping it. Research: CQGen generates comprehensive verification questions covering every aspect of the passage (6 few-shot demos, sampled 3x, union). Retrieve per question. Agreement model (few-shot CoT): state the implied answer from the passage AND from the evidence, then judge agree/disagree. Edit model (fires only on disagreement): identify the exact span to edit, minimally revise to agree with evidence — REJECT edits with edit distance > 50 chars or > 0.5x length (preservation guard). CoVe supplies the anti-self-contamination trick: answer verification questions in a FACTORED prompt that does NOT see the baseline draft, because 'models that attend to existing hallucinations in the context tend to repeat them' (independent verification ~70% vs ~17% in-draft accuracy); Factor+Revise adds a cross-check emitting CONSISTENT/INCONSISTENT and keeps only consistent facts for the final synthesis.

CITATION CERTIFICATION (ALCE/LongCite): certify at basket granularity with NLI. Recall = does the CONCATENATION of all cited spans entail the statement (TRUE T5-11B NLI). Precision = a citation is irrelevant iff it does not entail alone AND the remaining basket still entails without it. **Concrete methods:** The field has converged on TWO reference designs that BOTH decouple composition-granularity from verification-granularity — which is exactly POLARIS's missing move.

(A) OUTLINE-FIRST HIERARCHICAL — WebWeaver (the single closest architecture to POLARIS's need). Two agents over one shared evidence memory bank:
- PLANNER runs a ReAct loop with three actions: {search, write_outline, terminate}. `search` queries the web, LLM-filters URLs, and writes each retained source into the memory bank as an ID-indexed entry `id_k = {query-relevant summary, verifiable detailed evidence (quotes/data points)}`. `write_outline` is NOT one-shot — it is repeated refinement: after each search round the planner rewrites the WHOLE outline and populates every section/subsection with `<citation>id_x, id_y, id_z</citation>` tags mapping that section to specific memory-bank evidence IDs. This citation-grounded outline is the global-thesis + coherence carrier. It terminates when the outline is comprehensive.
- WRITER then walks the outline section-by-section with actions {retrieve, think, write, terminate}. For each section it (1) `retrieve`s ONLY the evidence IDs the outline pinned to that section (a basket), (2) `think`s over prior sections + retrieved basket, (3) `write`s the section prose inside `<write>` tags, then (4) PRUNES that section's source material out of context and replaces it with a placeholder before moving on. So each section is composed at SECTION scale over its whole evidence basket (cross-source synthesis), while traceability is guaranteed because the prose was written from a known, pinned set of evidence IDs.

(B) DRAFT-FIRST DIFFUSION — TTD-DR (the coherence-first counter-design). Treats report writing as denoising: an LLM writes a full preliminary report skeleton from the query alone (the "noisy draft"; global coherence exists at step 0), then a retrieval loop iteratively removes "noise" (imprecision/incompleteness). Each denoising step: generate a search question targeting a gap in the current draft, retrieve an answer, then REVISE the whole draft with that delta. Runs up to 20 steps. Layered on top is component-wise self-evolution: for each component (plan, search-query, answer, final report) sample multiple variants (temperature/top_k), score each with an LLM-as-judge on Helpfulness+Comprehensiveness, revise each toward higher fitness, then CROSS-OVER/merge the variants into one output. This is how it gets depth without losing the global thesis: the whole-report draft is never abandoned, only denoised.

(C) STORM (canonical predecessor of A): discover multiple PERSPECTIVES on the topic; each perspective runs a simulated multi-turn conversation asking questions to a retrieval-grounded "expert" (this is what drives breadth/coverage); curate an outline from collected references; write each section from outline+references; combine + polish sections into the final article.

The shared blueprint = retrieve → PLAN a global citation-grounded outline (or draft) → synthesize per-section over the pinned evidence basket → verify/edit as a downstream layer. Composition happens at section/report scale; provenance is carried by the outline-to-evidence-ID binding, not by composing one sentence per span.

**Exact details (formulas / steps / prompts / algorithms):** DEEPTRACE (arXiv 2509.04499) — 8 metrics, Equations 1-8 (as extracted; matrix ⊙ = element-wise product):
1. One-Sided Answer (Eq1) = 0 if BOTH pro and con statements present, else 1 (debate queries only).
2. Overconfident Answer (Eq2) = 1 if One-Sided=1 AND Answer-Confidence=5, else 0.
3. Relevant Statement (Eq3) = (#relevant statements)/(total statements).
4. Uncited Sources (Eq4) = (#cited sources)/(#listed sources).
5. Unsupported Statements (Eq5) = (#unsupported statements)/(#relevant statements).
6. Source Necessity (Eq6) = (#necessary sources)/(#listed sources), where "necessary" = a minimum vertex cover of the statement-source bipartite graph via Hopcroft-Karp.
7. Citation Accuracy (Eq7) = Σ(CitationMatrix ⊙ FactualSupportMatrix) / Σ(CitationMatrix). [Of the citations you MADE, what fraction are actually supported — this is the faithfulness precision metric.]
8. Citation Thoroughness (Eq8) = Σ(CitationMatrix ⊙ FactualSupportMatrix) / Σ(FactualSupportMatrix). [Of the support that EXISTS, what fraction you actually cited — recall.]
Pipeline: decompose answer into statements → tag each {query-relevant y/n, pro/con/neutral} → assign a 1-5 Likert Answer-Confidence via LLM judge (Pearson 0.72 vs humans). Build CitationMatrix (statements×sources, binary: did statement cite source). Build FactualSupportMatrix (same shape, binary: does source support statement) via GPT-5 judge (Pearson 0.62 vs manual; ~80,000 support evaluations). Judge = GPT-5. Reported: generative search engines 40-80% citation accuracy; GPT-5(DR) best at 79.1% accuracy / 87.5% thoroughness / 54.67% one-sided.

DRB-II (arXiv 2601.08536) — dimensions: InfoRecall (retrieve task-relevant info accurately+comprehensively), Analysis (synthesize into higher-level insight beyond aggregation — trends, causal/comparative reasoning, conclusions), Presentation (clear structure, tables/figures, transparent citation). 4-stage rubric build: (1) LLM extracts research task + binary rubrics from expert articles, rubrics must be ATOMIC/indivisible, encode target content directly, verify numerics with margins; (2) self-evaluate rubrics against source — regenerate any dimension scoring <90% accuracy; (3) human atomic/consistency revision; (4) 400+ expert-hours review across 22 domains. Judge = Gemini-2.5-Pro, binary 0/1 per rubric, batched 50/pass (best F1 89.57 @ batch 50). Per-task score = (#rubrics passed)/(#rubrics). Rubric distribution/task: InfoRecall 52.9, Analysis 12.8, Presentation 5.7 (~9,430 total / 132 tasks, 66 EN + 66 ZH). Blocked-reference: source articles blocklisted in prompt + reports referencing the source are excluded, forcing independent multi-source retrieval. Example rubric: "Presented the change in gold prices in table form." Leaderboard (Overall): OpenAI o3 DR 45.40 (InfoRecall 39.98/Analysis 49.85/Presentation 89.16); Gemini-3-Pro DR 44.60; Gemini-2.5-Pro DR 41.98. NOTE: no top system passes >50% of rubrics; overall ≈45 while Presentation ≈89 implies Overall is NOT a flat mean of the three — it is weighted toward InfoRecall+Analysis (exact weighting not stated in the extracted text — flagged as uncertain).

WEBWEAVER exact: DeepResearch-Bench RACE 50.58 (Claude-sonnet-4 writer) vs Gemini-2.5-Pro 49.71; citation accuracy 93.37% vs openai-deepresearch 75.01%; effective citations 200.75. Ablations: outline optimization 48.85→50.82 over 3 rounds; hierarchical vs brute-force writing: insight 42.72→50.02 AND citation accuracy 86.73→93.37. Agentic FT (Qwen3-30b-a3b on 3.3k plan + 3.1k write trajectories): citation accuracy 25%→85.90%.

TTD-DR Algorithm 1 (verbatim structure): for t in 1..N: Q_t = M_Q(q, plan, R_{t-1}, Q, A) [question at gap in draft]; A_t = M_A(Q_t) [retrieve]; R_t = M_R(q, R_{t-1}, Q, A) [remove noise/revise draft]; break on exit. N=20. Self-evolution hyperparams: n_q=5 query variants, n_a=3 answer variants, n_r=1-5 report variants. Answer-merge prompt: "Combine them into a single answer so that it best fulfills the initial user query; if conflicting, reconcile logically; only output a combined answer from <answer_list>, do NOT use other information." Results: LongForm win 69.1%, DeepConsult win 74.5%, HLE-Full 34.3 vs OpenAI DR 26.6, GAIA 69.1. Ablation: backbone 39.4 → +self-evolution 60.9 → +denoising 69.1 (LongForm).

**How it scores high:** DEEPTRACE (faithfulness — POLARIS's strength, must stay #1): the winning move is that outline-first / draft-first systems make Citation-Accuracy (Eq7) high WITHOUT atom-composition. WebWeaver hits 93.37% because the writer only ever writes a section from the exact evidence basket the planner pinned to it, so almost every citation lands inside its supporting span — high Σ(Citation⊙Support)/Σ(Citation). POLARIS already maximizes Eq7 via strict_verify/NLI/D8 (a supported-only gate). The DANGER metric is Citation THOROUGHNESS (Eq8, recall): if POLARIS drops or fails-open a sentence whose support exists but wasn't cited, thoroughness falls. Consolidating baskets (keep ALL corroborating sources per claim, multi-citation) directly raises Eq8. One-Sided (Eq1) + Overconfident (Eq2) are won by deliberately composing BOTH pro and con statements per contested claim and not maxing confidence language — an outline/analysis behavior, cheap to add. Unsupported-Statements (Eq5) and Source-Necessity (Eq6) are already POLARIS strengths (per-sentence verification + weighted multi-source baskets).

DRB-II (coverage+analysis — POLARIS's weakness, the whole reason for this research): InfoRecall (52.9 rubrics/task, the dominant weight) rewards breadth — it is won by STORM/WebWeaver-style multi-perspective planning + blocked-reference-robust independent multi-source retrieval, NOT by faithfulness. This confirms the operator memory that coverage must be its OWN workstream: no faithfulness gate can add an InfoRecall rubric hit for a fact you never retrieved. Analysis (12.8 rubrics/task) is won ONLY by cross-source synthesis — trends, causal/comparative reasoning, conclusions that "go beyond simple aggregation" — which is EXACTLY what POLARIS's atom-composition + source-by-source list structurally cannot produce (a per-source list scores ~0 on Analysis rubrics). WebWeaver's ablation (insight 42.72→50.02 from hierarchical section-writing over baskets) and TTD-DR's whole-draft denoising are the mechanisms that generate Analysis-rubric-satisfying prose. Presentation (5.7 rubrics/task, table/figure/structure) is won by the global outline + rendering tables — the smallest lever but where top systems already score ~89.

**Reference system designs:** WebWeaver (closest to POLARIS): planner-writer over an ID-indexed evidence memory bank; iteratively-refined citation-grounded global outline as the coherence spine; writer does per-section targeted retrieval of ONLY the pinned evidence IDs + retrieve-and-prune context management (each finished section's sources are evicted and replaced by a placeholder). Failure modes it mitigates: (a) citation hallucination — mitigated by binding citations at PLAN time so the writer synthesizes from a known basket (93.37% acc); (b) long-context rot / cross-section interference — mitigated by prune-after-write; (c) brute-force "dump everything then write" incoherence — mitigated by hierarchical section-scoped synthesis.

TTD-DR (coherence-first): full-report skeleton draft first, then <=20 retrieval-denoising revisions of the whole draft + component-wise self-evolution (variant sampling → LLM-judge fitness on Helpfulness/Comprehensiveness → revise → cross-over merge). Failure mode mitigated: "information loss" and disjointed islands from bottom-up assembly — the draft-centric loop keeps a single coherent artifact throughout; its stated weakness is self-denoising without retrieval converges slowly/sub-optimally, hence retrieval is the mandatory denoising signal.

STORM: multi-perspective question-asking + simulated grounded-expert conversations to force BREADTH, then outline curation, then section writing + combine/polish. Documented failure modes (from its own Wikipedia-editor eval): source-bias transfer and over-association of unrelated facts — relevant warnings for POLARIS's consolidation step.

OpenAI Deep Research / Gemini Deep Research (the DRB-II + DeepTRACE leaderboard incumbents): agentic plan-search-synthesize loops; empirically they TRADE OFF the two axes — DeepTRACE shows GPT-4.5/Perplexity are highly one-sided (83-90%) and only 40-50% citation-accurate, i.e. fluent-but-unfaithful; DRB-II shows even o3-DR passes <50% of rubrics with Analysis ~50. No shipped system is simultaneously top on faithfulness AND coverage/analysis — that gap is POLARIS's opening.

**Applicability to POLARIS:** POLARIS's confirmed root — composition at ATOM granularity (one sentence certified against one span) + additive fail-open coverage tail = choppy prose + source-by-source list + disconnected islands — is precisely the anti-pattern both reference designs abandoned. The architecture-level fix, honoring the hard rule that faithfulness (strict_verify/NLI/D8/provenance) is NEVER relaxed:

1. DECOUPLE composition granularity from verification granularity. This is the central lesson. WebWeaver/TTD-DR compose at SECTION/REPORT scale and treat citation-support as a SEPARATE downstream layer; POLARIS made the verification unit (one sentence↔one span) into the composition unit. Keep per-sentence verification as-is, but stop composing at that granularity.

2. ADD a WebWeaver-style planner producing a global, citation-grounded OUTLINE that binds each section to a BASKET of evidence IDs — POLARIS already HAS baskets (the consolidation layer) and IDs (provenance tokens). The outline is the missing global-thesis/coherence carrier and it is faithfulness-NEUTRAL (structure only, no claim generation).

3. COMPOSE each section over its WHOLE basket as connected cross-source prose (the WebWeaver writer: retrieve pinned basket → think across sources → write section), NOT atom-by-atom. This is what generates DRB-II Analysis-rubric content (trends/causal/comparative) and kills the source-by-source list.

4. VERIFY per-sentence AFTER composition as a gate: run strict_verify/NLI/D8 over the composed section; drop/regenerate failing sentences. Because the section was written FROM the correct pinned basket, expected drop rate is low (WebWeaver's 93.37% is the empirical analogue). Faithfulness is thus UNRELAXED and even strengthened by basket-faithfulness (verdict against the whole basket, per §-1.3).

5. Replace the additive fail-open coverage tail with a TTD-DR-style whole-report skeleton draft at step 0 (guarantees a global thesis exists before section-fill), then denoise sections with retrieval — this removes the "islands" failure directly.

6. Coverage/breadth (DRB-II InfoRecall) MUST be a co-equal STORM-style multi-perspective retrieval workstream (per operator memory 2026-07-05): a faithfulness gate can never add a fact you didn't retrieve. Add pro/con dual-statement composition per contested claim to win DeepTRACE One-Sided/Overconfident cheaply.

Net: a retrieve → plan-outline(basket-bound) → section-synthesize-over-basket → per-sentence-verify-gate → global-edit pipeline. It is SURGICAL per §-1.3: reuse existing baskets, provenance IDs, and the faithfulness engine; add an outline planner + section-scale writer; delete the atom-composer and the additive coverage tail.
