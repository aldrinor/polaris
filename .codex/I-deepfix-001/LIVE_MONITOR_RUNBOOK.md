## Tick design
Every 5 minutes, one tick = a stage-ordered, section-grouped FORENSIC read of every NEW log line since the saved byte-offset (per box: /workspace/paid_deep_wave3.log resp. /workspace/paid_fresh3.log; tail from offset, save new offset). Structure of each tick, in this exact order:

(0) LIVENESS + STAGE FIX: PID alive (pgrep -f run_gate_b.py), traceback/faulthandler scan (grep -E 'Traceback|CUBLAS|CUDA out of memory|MemoryError'), and detect the CURRENT pipeline stage from the newest stage-marker lines (fetch lines / '[resume] A15' / 'judge progress' / outline / '[contract_section]' / 'verified-compose PRIMARY' / summary_table canary / 'WS-1(b)' / run_validity_gate). Stage regressions or 5+ min of zero new lines during an LLM stage = stall flag.

(1) FETCH YIELD FIRST — ALWAYS the first number printed, every tick, even mid-compose (operator hard rule 2026-07-07): fetched=N / attempted=M + failure/timeout counts from the retrieval telemetry, plus the gate marker grep '\[activation\] fetch_yield_gate:' expecting 'rate=X floor=0.30 -> pass'. HARD HALT RULE: if yield < floor and the gate line says HALT, confirm the run actually stopped; if yield is bad and compose-stage lines appear anyway (gate bypassed/inert), KILL IMMEDIATELY — a starved corpus banked is the one failure a resume can never undo. On the resume box, tick 1 additionally requires the A15 block (flagged=N / attempted / recovered) as the yield-equivalent number.

(2) STAGE-ORDERED FIX SWEEP: run each fix's green/red greps from fix_signals, but only for fixes whose stage is at-or-behind the current stage. INERTNESS CHECK on every stage ENTRY: within the first tick after a stage's precondition traffic exists, that stage's activation markers MUST appear (flag ON + traffic + no marker = RED, the flipped-flag-silent-no-op case). This is what catches a mis-fire AT its stage instead of post-render.

(3) SECTION GROUPING: build the tick table per SECTION using the section-carrying markers — '[multi_section] <Section> verified-compose PRIMARY: N baskets -> draft_chars=M', per-section 'compose_offtopic_basket_screen: withheld=N kept=M', per-section basket-scaled-wall / WALL-DEADLINE / recovery-pass lines, per-slot '[contract_section]' lines (N2/N4/N5/B5 name their slot), per-section verification_details as each section completes. A red therefore names its section the moment its stage runs — never a post-render autopsy.

(4) CONTENT READ, NOT STATUS (operator rule 2026-07-01, §-1.1 quality): every tick, read actual CONTENT samples from the new material — newly fetched source titles + tier mix (credible non-journals are FINE at weight; junk domains are the tell), one extracted body head (prose or furniture?), the newest completed section draft's first paragraph claim-vs-cited-span, the newest Boundary/Tension line, gap sentences verbatim. Surface ANY problem seen, including non-current-fix issues — most SOTA-blocking signal is in the live logs BEFORE render.

(5) CONSOLIDATED TICK REPORT to the operator in plain spoken English: fetch-yield number first, then current stage, then a per-fix green/red table (14 rows), then the per-section table, then content-read findings, then spend/429 count. Short sentences.

(6) ON ANY RED: immediate Codex+Fable escalation (the 2026-07-04 military order) with the exact log lines attached; they decide hold/fix/relaunch-from-nearest-checkpoint vs let-run; Claude executes. Never wait for the next tick to escalate a red.

## Kill vs let-run
KILL NOW (stop the process this tick, before more money burns or a checkpoint banks poison): (1) Fetch yield below PG_MIN_FETCH_YIELD=0.30 with the gate NOT halting, or ANY sign a starved/chrome-fed corpus is about to bank (B1/B2 inert during fresh fetch; A15 disabled/zero-flagged on resume) — the corpus is the one artifact a resume can never repair. (2) 'Fourth Industrial Revolution' text in any query, plan, or draft — deterministic march to abort_run_validity_gate; killing at minute 30 saves the whole generation+D8 spend. (3) A stage-entry inertness RED on a run-shaping fix: no compose_offtopic_basket_screen marker at first section compose, no basket-scaled-wall line at first pre-pass, no FINDING#5 legacy-outline strip line on the legacy branch — kill before that stage completes, fix env/commit, resume from corpus_snapshot (env flags cannot hot-change in a live process, so 'fix live' for env-inert flags ALWAYS means kill+resume-from-nearest-checkpoint). (4) WALL-DEADLINE mass abandon >30% of a section's baskets with no recovery pass. (5) kept=0 from the off-topic screen in any section (over-withhold — an all-gap section).
