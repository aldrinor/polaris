## real_architecture

## real_architecture

End-to-end: PLAN -> RETRIEVE -> CONSOLIDATE -> SYNTHESIZE (coherent prose FIRST) -> ATTRIBUTE (coarse-to-fine) -> VERIFY (per-sentence, downstream) -> REPAIR -> RENDER. The crisp reconciliation (brief A4 + pack 5): **global coherence is produced ONCE, at COMPOSE time, over basket groups and the cross-basket section plan; per-sentence verification runs DOWNSTREAM as a filter that drops/hedges/re-attributes a sentence but never re-fragments prose and is never the compositor.** Composition granularity (paragraph/section) is decoupled from certification granularity (sentence) — the exact split arXiv 2604.01432 shows raises attribution F1 16-97% while curing choppiness, and the split WebWeaver/TTD-DR/LongCite all converge on.

1. **PLAN.** The section<->evidence binding already exists: `_section_baskets_for_compose(section, credibility_analysis)` at `multi_section_generator.py:4927` is POLARIS's WebWeaver "outline node -> citation IDs" map. Upgrade it to carry (facet, basket-ids, pro/con allocation) per section, fed by the facet tree from `expert_facet_planner.py` (its `counter_evidence` lens at :44 is the STORM pro/con perspective).
2. **RETRIEVE.** STORM retrieval + tier classifier stay as-is (weight-not-filter). Frontier widened by `PG_EXPERT_FACET_PLANNER` (`expert_facet_planner.py:56`) + `PG_SUBENTITY_QUERY_EXPANSION` (`sub_entity_query_expander.py:62`): ~35 facet/perspective queries instead of 7.
3. **CONSOLIDATE.** `credibility_pass.py` with `PG_BASKET_CONSUME_FINDING_DEDUP` ON (:65-76): baskets consume finding_dedup's ~99 multi-member clusters instead of claim_graph's ~1781 singletons, so multi-origin baskets (verified_support_origin_count>=2) actually exist. Keep-all, merge-only.
4. **SYNTHESIZE — coherence is born here, nowhere downstream.** Per section, the writer (`abstractive_writer.py`) drafts a coherent multi-sentence narrative per multi-source basket GROUP; cross-basket analytical units (`cross_source_synthesis.py` with plan-based pairing) and the section lead are drafted in the same pass. LongCite compose-first ordering: fluent prose exists BEFORE any verification verdict shapes it.
5. **ATTRIBUTE coarse-to-fine.** The writer emits per-sentence `[#ev:id:start-end]` tokens at generation time (generation-time attribution, high precision); on a bind failure, LongCite CoF re-attribution: coarse = candidate basket member row, fine = snap to the sentence-span via the existing PG_SPAN_RESOLVER re-anchor (verify_sentence_provenance re-points the token; consumed at `verified_compose.py:1286-1289`).
6. **VERIFY — the per-sentence gate, downstream only.** UNCHANGED strict_verify + NLI entailment + the STRICTER writer wrapper `make_writer_verify_fn` (`abstractive_writer.py:285-341`: no local-window rescue, judge_error fail-closed, span->sentence numeric completeness) + own-region gate `_tokens_within_basket_regions` (`verified_compose.py:1293`) + 4-role D8 seam + report_redactor. A failing sentence routes to REPAIR, not to concatenation.
7. **REPAIR.** Bounded RARR-style re-draft inside `_compose_one_basket` (`verified_compose.py:1281-1305`): failure_reasons feed the writer's existing `revise_reasons` plumbing (`_build_writer_prompt`, `abstractive_writer.py:362-400`); the writer re-drafts the WHOLE paragraph so prose stays connected; after `PG_WRITER_REPAIR_MAX` tries the still-unverified fact ships as a LABELED verbatim K-span disclosure quote, never as body prose.
8. **RENDER.** [N] resolution, the corroboration WEIGHT channel (CWF `_basket_corroboration_block`), presentation tables, ONE chrome screen (`is_render_chrome_or_unrenderable`).

## traceability

## traceability

1. **Inline citation set per sentence = all-and-only INDEPENDENTLY-ENTAILING members.** New `src/polaris_graph/generator/citation_set_minimizer.py` behind `PG_MIN_CITE_SET` (default OFF). Post-verify, per sentence, run the ALCE leave-one-out policy over its basket members with the existing entailment judge as phi: PRUNE from the inline cite any member whose span does not itself entail THAT sentence (these tank Citation-Accuracy = Sum(C.F)/Sum(C)); KEEP every member that independently entails it (each is a true supporter — protects Citation-Thoroughness = Sum(C.F)/Sum(F), the metric that REWARDS multi-citation baskets); members that are report-level MVC-redundant (identical statement-set coverage, the Source-Necessity Hopcroft-Karp penalty) demote from inline cite to the weight channel. Honest tradeoff stated up front: a strictly-minimal set would HURT Thoroughness, so the policy is prune-non-entailing + demote-only-MVC-redundant — never prune a genuine independent supporter of that exact sentence.
2. **Corroboration WEIGHT channel (keep-all reconciled with Accuracy/Necessity).** The REST of the consolidate-keep-all basket — members that entail the CLAIM/basket but not this exact sentence, plus MVC-demoted members — renders in the existing CWF corroboration surface (`_basket_corroboration_block`, the I-wire-014 FIX-A header machinery) as count + tier weights ("corroborated by 5 sources: T1x2 T2x2 T4x1"), NOT as inline [N] citations. This is exactly the §-1.3 "weight surfaced to the user, never a citation" resolution from brief A6.3: nothing is deleted, the basket disclosure JSON keeps every member, and the DeepTRACE judge never sees a cited-but-non-entailing source.
3. **LongCite coarse-to-fine attribution on bind failure.** Coarse = candidate basket member rows (the 128-token-chunk analog); fine = snap to the sentence-span [a-b] inside the row via the existing PG_SPAN_RESOLVER re-anchor (verify_sentence_provenance re-points the token — consumed at `verified_compose.py:1286-1289`). Only after CoF re-attribution fails does the sentence enter the bounded repair loop, so re-attribution is always tried before re-drafting.
4. **Two-sided pro/con synthesis for debate queries** (One-Sided/Overconfident — even GPT-5 DR sits at 54.67 one-sided; the field's hardest metric). Plan-time debate-query detector; the `expert_facet_planner` `counter_evidence` lens (:44) guarantees con-side retrieval reaches the corpus; composition places >=1 verified pro AND >=1 verified con clause joined via the licensed "conflict" connective (`LICENSED_CONNECTIVES["conflict"] = "; in contrast, "`), and the writer contract's hedge-preservation keeps certainty language honest (kills Overconfident = One-Sided AND confidence 5). If the corpus genuinely carries no counter-evidence, the section DISCLOSES the one-sidedness honestly — balance is never fabricated.

## faithfulness_approach

## faithfulness_approach

The ONLY hard gate remains strict_verify + NLI entailment + 4-role D8 + provenance tokens + span-grounding (`provenance_generator.verify_sentence_provenance` / `strict_verify`) — byte-untouched, clinical-safety-critical, NEVER relaxed. It moves from being the COMPOSITOR (the confirmed root cause) to being a DOWNSTREAM per-sentence FILTER whose verdicts route to drop / hedge / re-attribute / repair. No flag in this design touches a verification threshold, judge, gate condition, or abort semantic (§9.1.4 zero-verified-abort unchanged).

Specifics: (1) The writer path keeps its STRICTER-than-baseline wrapper unchanged — `make_writer_verify_fn` (`abstractive_writer.py:285-341`): `allow_local_window_fallback=False`, transport judge_error forced `is_verified=False` fail-closed, span->sentence numeric-completeness escalation. The bounded repair loop changes only WHICH draft is submitted for verification, never how any draft is judged; every repair attempt re-runs the full unchanged gate. (2) Cross-source/analytical units verify per CLAUSE against each clause's OWN basket-scoped pool + own-region gate (P1-1/P1-2, `_tokens_within_basket_regions` at `verified_compose.py:1293`) — never one aggregate sentence verified against a union pool. Connectives carry no token and are licensed only by certified engines (ContradictionEdges, `entails_directional`, the new deterministic numeric comparator), all failing CLOSED to neutral; `relational_quantifier_guard` still strips unlicensed aggregate predicates. (3) D8 + report_redactor still drop settled non-VERIFIED prose; depth findings still ride the D8 seam (`PG_DEPTH_SYNTHESIS_D8_GATE`). (4) The K-span disclosure fallback is verbatim and trivially re-passes the gate; it is labeled, never promoted to body prose. (5) Basket faithfulness is STRENGTHENED per §-1.3.3: with multi-origin baskets real, each claim's verdict now carries genuine corroboration (count + weights + agreement) instead of a single-URL blind spot. The clinical rule holds at every seam: under-relax is safe; over-relax is lethal; nothing in this design over-relaxes.

## biggest_risk

## biggest_risk

**The bounded repair loop fails to converge under the strict writer wrapper on real corpora.** If most baskets exhaust PG_WRITER_REPAIR_MAX (the wrapper is deliberately harsher than baseline: no local-window rescue, judge_error fail-closed, numeric completeness — and GLM paraphrases that move a number or lose 2-word overlap fail exactly as depth_synthesis's free-redraft does today, returning [] at `depth_synthesis.py:107-109`), the body silently degenerates back to atom concatenation wearing a "disclosure quote" label: PG_SYNTH_PRIMARY reads ON while the primary path is dark. This is the exact partial-fired failure mode the project has hit repeatedly (14-winner wiring ~1/8 firing; I-wire-013 fixes committed+gated+ran but absent from the render).

**Concrete de-risk:** (1) Replay harness FIRST — measure the per-basket repair-convergence distribution on 3 banked corpora BEFORE any paid run; if more than ~20-30% of baskets exhaust the budget, iterate the writer prompt and the CoF re-attribution step (never the gate) until convergence is boringly high; the revise_reasons feedback already names the exact wrapper failure, so the prompt can be tuned against real reasons. (2) Fail-loud dark-path canary in `run_gate_b.py`: FAIL the run when PG_SYNTH_PRIMARY is ON and zero sections shipped writer-path prose — the same accepted class as `assert_cross_source_synthesis_fired` (a silent-no-op detector, not a target). (3) Per-section compose-time log of writer-kept vs K-span-fallback vs disclosure counts, so the 5-minute forensic read catches degeneration live, mid-run, before render. (4) Ship V1/V2/V3 variants so a composition-convergence failure is isolated from the coverage changes, and the labeled K-span fallback preserves the clinical always-release property while the loop is tuned — the failure mode is visible and honest, never a silent fake success.
