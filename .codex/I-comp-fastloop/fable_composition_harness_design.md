## 5. Fast-loop UX

- **Profiles.** `--profile pipeline` (DEFAULT, honest full fidelity): per case, write the subset snapshot to `<case_out_root>/<domain>/<slug>/corpus_snapshot.json`, invoke `run_gate_b.py --only <slug> --resume --out-root <case_out_root>` as a subprocess (the sanctioned launcher applies the full capability slate ‚ÄĒ run_honest_sweep_r3.py:20600-20614), then run the oracle. Includes real LLM spend: outline + section drafts + depth/analyst synthesis + D8 4-role ‚ÄĒ unavoidable, but small because it scales with the ~8-60 row case. Estimated 10-25 min/case. `--profile render` : delegate to `iwire013_fast_render_audit.py` (offline, ~1-2 min, no LLM) for render-side fixes; D8 + writer legs stamped "NOT RUN (render profile)" so a pass cannot over-claim.
- **Parallelism.** Cases run as parallel subprocesses, bounded `PG_COMPOSE_HARNESS_MAX_PARALLEL` (default 2, LLM-network-bound; ¬ß8.4: one Codex at a time is unrelated, but keep ‚Č§3 pipeline children and kill children on total-deadline breach). Deterministic case ordering = YAML order; results.json ordered the same. LLM prose is not byte-deterministic ‚ÄĒ the ORACLE is deterministic given the report, and structural defects reproduce; stated honestly in the report header.
- **Selection.** `--only <case>` (one case, ~10-25 min), `--rerun-failures <results.json>`, `--list`, `--bridge <bodies.json>`, `--case-file <yaml>`.
- **Flag-gate refusal.** At startup assert ALL 14 I-deepfix-006 flags ON (each module's `*_enabled()` reader), plus `PG_STRICT_VERIFY_ENTAILMENT=enforce` for the pipeline profile and OPENROUTER key present. Any OFF ‚áí print "RESULT VOID ‚ÄĒ FIX FLAGS OFF", write no PASS, exit 2. Flag states stamped into report.md header + results.json (fetch-harness `check_flags` pattern).
- **Outputs.** `outputs/compose_harness/<utc>/results.json` + `report_index.md` + per-case `<case>/report.md` (the actual prose for the ¬ß-1.1 read-aloud) + the oracle findings with quoted spans. Exit codes: 0 green, 1 any FAIL/UNREACHABLE, 2 VOID, 3 internal/canary-red ‚ÄĒ identical semantics to the fetch harness.
- **Where.** VM for the pipeline profile (all heavy LLM runs on the VM per standing rule; .env with OPENROUTER + Zyte already deployed there). Render profile runs anywhere (offline).
- **Realistic chained runtime.** fetch harness ~5 min ‚Üí bridge (seconds) ‚Üí compose harness 3 cases in parallel ~15-30 min wall ‚Üí ¬ß-1.1 read. Versus the 3-6 h full pipeline: the fix‚Üítest cycle drops from "overnight" to "under half an hour", and render-only fixes iterate in ~2 min.
