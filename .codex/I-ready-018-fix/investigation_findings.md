### root_cause
render_slot_prose (slot_fill.py:674-732) renders an EMPTY frame slot as PROSE. A required field that the source genuinely does not support (status='not_extractable') is emitted as a grammatical, citation-bearing sentence that asserts nothing ('Journal quality tier: not extractable from available primary content [N].'). These sentences are contentless by construction yet pass every per-sentence faithfulness check (valid ev id, no numbers to mismatch, label-word overlap with span), so strict_verify keeps them, the rescue path protects them (contract_section_runner.py:124-125), the regroup writes them into report.md body, and Gate-B promotes each to a 4-role claim. The design intent (M-58 docstring + M-68 Fix #1) was 'no silent omission of a slot' — but it over-corrected into 'verbalize every unfillable field,' producing per-FIELD placeholder spam rather than per-SLOT honest disclosure. The correct unit of disclosure is the slot (and it already exists: the zero-kept gap path at contract_section_runner.py:945-1011 emits ONE honest 'curator-actionable gap' sentence, as seen for Frey-Osborne at report.md:30). The bug is that partial slots and the per-field render path bypass that and emit per-field 'not extractable' prose instead. Net: a generation-degeneracy / template-leak, not a hallucination — the report is contentless-but-honest, which is exactly why coverage collapsed to 0.286 and the gate correctly HELD.

### proposed_fix
Cut the placeholder at the single source — render_slot_prose (slot_fill.py:674-732) — so unfillable slots DROP instead of verbalizing. Concretely: (a) in render_slot_prose, SKIP fields with status=='not_extractable' (and the mixed gap_unrecoverable branch at 728-731) — do not append a sentence for them; render only status=='extracted' fields. (c) The honest gap disclosure is NOT lost: it remains in manifest.frame_coverage_report (failure_reason='no substantive verified prose — only not-extractable disclosures', status=curator_gap_no_substantive_content) and human_gap_tasks.json — non-scored audit surfaces. This is the minimal cut: one edit upstream of strict_verify, the rescue exemption (124-125), and claim extraction, so the placeholders never enter ANY of those paths. ACCEPTANCE CRITERION must be 'drb_72 STILL HOLDS (release_allowed=False)' — NOT 'coverage improves.'
