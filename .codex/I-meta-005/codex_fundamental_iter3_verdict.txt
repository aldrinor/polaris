**The diagnosis is a spectrum, all three points verified by reading code/source:**

### Verification (gaps 15, 16, 18 — keeping 13/14/17/19)
The monolithic per-sentence check is replaced by a **claim-type router → three regimes**, additive-only and fail-closed. The router defaults to the strict path; any atom with a number/decimal/unit/named-entity/**qualitative-negation** is force-routed to the strict/computation path regardless of classifier opinion.
- **Regime A — factual (UNCHANGED guarantees):** today's `verify_sentence_provenance` numeric-verbatim + span-bounds + ≤400-char local-window + ≥2 content-word overlap + named-study identity + entailment-on-bounded-window. Preserves the 2026-05-30 composition-fix lock (whole-doc widening = candidate generator only; binding verdict re-judges the short span; fail closed). #13/#14 intact.
- **Regime B — analytical (gap 18, NEW):** decompose the sentence into atoms (VeriScore verifiable-claim extraction + decontextualization https://aclanthology.org/2024.findings-emnlp.552/ ); each factual atom verified by Regime A against a **single tight span**; the integrative connective licensed iff its atoms passed; MiniCheck/Granite entailment (https://aclanthology.org/2024.emnlp-main.499/ ) on the **concatenated cited spans of joined atoms only** as a *rescue* signal — union/whole-doc may propose a candidate window but never binds the verdict. **ALCE citation-precision guard** (https://aclanthology.org/2023.emnlp-main.398.pdf ): every cited span must be load-bearing (removing it changes ≥1 atom verdict) — stops "cite 20, union covers everything" laundering. This rescues policy reasoning that the single-span check wrongly dropped, without relaxing any FAIL. Backbone: SAFE/FActScore atomic decomposition https://arxiv.org/abs/2403.18802 , https://aclanthology.org/2023.emnlp-main.741/
- **Regime C — modeled-vs-sourced (gap 15, NEW):** computed numbers carry a `[#calc:<model_id>:<field>]` (a.k.a. `[#model:…←ev_ids:modeled=vars]`) token; verified against the **computation record**: output matches the executed script verbatim; every *input* is itself Regime-A-grounded; formula id is in an audited registry; the "(modeled assumption)" label is *enforced* (hard drop if missing). Routing signal, not just a display label — this prevents strict_verify from gutting gap 9's own output.
- **Gap 16 — qualitative conflict:** `(did not|no|never|absent|without|failed to)\s+…\s+<endpoint>` atoms force-routed to entailment (never bare overlap — "did not reduce" vs "reduced" share every content word: the lethal flip per 2026-05-26 memory); detected conflicts passed into the section prompt as **must-surface**; eval gate raises an unresolved-conflict warning when corpus holds both polarities and prose asserts one.

**Field-agnostic generalization of verifier helpers (generalize false-drop-preventers, never loosen a FAIL):** `_DOSE_PATTERN_RE` → typed-quantity primitive (`<number><unit>`, SI+currency+domain units, so "200 MW"/"C$2.4 bn"/"2.4 mg" all read as identifiers); `_PLACEBO_COMPARATOR_RE` → structural comparator primitive ("X versus/compared-with Y"); `_THRESHOLD_RE` → generic threshold primitive; trial-name gate → named-study/entity identity primitive (SURMOUNT-1≠SURMOUNT-3 lock stays). ## 5. THE QUANTIFIED-TRADE-OFF BUILD (gap 9 — the one real new capability)

**Critical finding (re-verified this session):** the substrate is ~70% built and DISABLED on the production path. `tools/code_executor.py` is a full **PAL** runtime (LLM writes Python; AST-validated import allowlist numpy/pandas/scipy/math; reflection blocklist; socket-kill preamble; restricted-env subprocess; 30s timeout; retry-on-error at `generate_and_execute_analysis()`). `tools/tool_registry.py` has `execute_python`/`comparison_table`/`meta_analysis`, each returning `source_evidence_ids` with per-item `[CITE:ev_xxx]`. `tools/react_agent.py` orchestrates them. But `run_honest_sweep_r3.py` **never imports** `react_agent`/`build_default_registry`/`code_executor` — it calls `generate_multi_section_report` + the unverified `analyst_synthesis` only. So gap 9 is a **re-wire + provenance-binding + faithfulness-wedge** problem, not greenfield. (PAL: Gao et al. ICML 2023 https://arxiv.org/pdf/2211.10435 , https://github.com/reasoning-machines/pal ; on-prem break-even template https://arxiv.org/pdf/2509.18101 ; FinSphere tool-augmented analyst https://arxiv.org/pdf/2501.12399 )

**Design — Extract → Model → Execute → Bind → Verify:**
1. **Extract (sourced numbers only):** reuse `evidence_extractor.extract_numbers_from_evidence()`; each number carries its `evidence_id`. A number with no `evidence_id` is NOT eligible as a model input — fail-closed.
2. **Model (declarative spec):** Writer (DeepSeek-V4-Pro) emits JSON `{title, inputs:[{name,value,unit,ev_id} | {name,sweep:[…],modeled:true}], formula, outputs, sensitivity_over}`. Every input is either **sourced** (`ev_id`) or explicitly **`modeled:true`** — no third category.
3. **Execute (deterministic PAL):** hand the spec to `code_executor.generate_and_execute_analysis()`; computes outputs + a sensitivity table (nested sweep) + a break-even solve (`scipy.optimize.brentq`, already on the allowlist). Arithmetic is *computed*, never *generated* — killing the gap-14 failure mode at its source.
4. **Bind (provenance on computed numbers):** computed outputs get `[#calc:<model_id>:<field>]` / `[#model:tco_sovereign←ev_017,ev_021:modeled=util_pct]` declaring which **cited inputs** fed it and which were **modeled assumptions**. The spec + executed script + dependency graph persist to the signed bundle (`outputs/.../quantified_model.json`), so audit-replay (#19) extends to the computation: a reviewer re-runs the script on the same cited inputs and reproduces the number.
5. **Verify (extend strict_verify, Regime C):** a computed sentence's `ev_id`s must each exist *and* have been a declared input; the asserted number must equal the executed output (±float tolerance); every `modeled=` variable must appear with a "(modeled assumption)" label or the sentence is dropped.

**How it stays in the wedge:** a fabricated computed number fails the equality check; an unsourced input fails Step 1; an unlabeled assumption fails Step 5. The wedge now covers computed artifacts — strictly stronger than the frontier. Honest scope (LAW II): POLARIS models a line item only where the corpus supplies inputs; otherwise it marks `modeled` (labeled) or honestly reports it cannot model that line — which is a WIN over Gemini's invented "60,000 GPUs / $9B," not a weakness. Gate behind `PG_ENABLE_QUANTIFIED_ANALYSIS`, default off until blinded-validated.

## 7. SIX-LINE PLAIN-SPOKEN SUMMARY (for the blind operator)

1. The core idea: stop hand-listing "good websites." Score every source live by what it is — who cites it, is it the original primary source, do independent sources agree — so it works for any field and any country with no list to maintain.
2. We delete the 1,871-line clinical website list and replace it with that computed score. The guarantee we didn't just build a new list isn't one grep — it's a four-part rule: no website names live in the code at all (they're in auditable data files), every source carries a confidence score so thin-coverage fields are flagged honestly instead of mislabeled, and we test hard cases on purpose.
3. The question drives everything: one planner reads the question, picks the sub-questions and the report's own sections, so a housing question never gets an "Efficacy" heading again.
4. The big new capability is real number-crunching: POLARIS writes and runs actual Python to compute cost break-evens and sensitivity tables, and every output number must trace back to a sourced input — so it can't invent a "$61M" figure.
5. The thing that makes us better than ChatGPT and Gemini — every number must match its cited source, it refuses when unsure, and the whole run replays — is kept untouched and now also covers the computed numbers.
6. Nothing costs money until the very last phase; every earlier phase has an offline pass/fail test first, and Codex reviews the two foundation pieces before we build on them.
