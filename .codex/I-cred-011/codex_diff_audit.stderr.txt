+"""I-cred-011 (Phase 11) — calibration metrics (Brier / ECE / MCE / reliability bins) — pure offline.
+
+A standalone, deterministic analysis utility for the credibility-weighted sourcing redesign: given
+(predicted_confidence, outcome) pairs — e.g. the per-claim ``certainty``/``credibility_weight`` POLARIS
+disclosed vs the ground-truth (1 = the claim held up in the §-1.1 line-by-line audit, 0 = it did not) —
+compute how well-calibrated those confidences are:
+
+  * ``brier_score``  = mean squared error between predicted probability and outcome (lower = better).
+  * ``ece``          = Expected Calibration Error: bin by predicted probability, average the per-bin
+                       |mean_predicted − mean_outcome| gaps weighted by bin population.
+  * ``mce``          = Max Calibration Error: the worst single-bin gap.
+  * ``reliability_bins`` = the per-bin breakdown (the reliability-diagram data).
+
+This is PURE offline math — NO network, NO LLM, NO production caller (it analyses runs post-hoc, so it
+needs no default-OFF flag for byte-identity). It does NOT touch any faithfulness gate. The adversarial
+vaccine benchmark + the §6b competitor head-to-head + the paid beat-both RUN that CONSUME these metrics
+are the spend-bearing follow-up (operator-budget-gated); this module is the cash-free scoring core.
+LAW VI: the bin count is env-overridable; snake_case; explicit imports.
+"""
+from __future__ import annotations
+
+import os
+from dataclasses import dataclass
+
+_ENV_BINS = "PG_CALIBRATION_BINS"
+_DEFAULT_BINS = 10
+
+
+def _int_env(name: str, default: int) -> int:
+    try:
+        value = int(os.environ.get(name, "") or default)
+    except (TypeError, ValueError):
+        return default
+    return value if value > 0 else default
+
+
+def _clamp01(value: float) -> float:
+    return 0.0 if value < 0.0 else 1.0 if value > 1.0 else value
+
+
+def _coerce_pairs(pairs) -> list[tuple]:
+    """Keep only numeric (predicted, outcome) pairs; clamp predicted to [0,1] and the outcome to {0,1}.
+    Non-numeric / malformed pairs are dropped (a calibration metric over garbage is meaningless)."""
+    out: list[tuple] = []
+    for pair in (pairs or []):
+        try:
+            predicted, outcome = pair
+            p = _clamp01(float(predicted))
+            o = 1.0 if float(outcome) >= 0.5 else 0.0
+        except (TypeError, ValueError):
+            continue
+        out.append((p, o))
+    return out
+
+
+@dataclass
+class ReliabilityBin:
+    lower: float
+    upper: float
+    count: int
+    mean_predicted: float
+    mean_outcome: float
+    gap: float            # |mean_predicted - mean_outcome|
+
+
+@dataclass
+class CalibrationReport:
+    n: int
+    brier_score: float
+    ece: float
+    mce: float
+    bins: list
+
+
+def brier_score(pairs) -> float:
+    """Mean squared error between predicted probability and 0/1 outcome (0.0 for no valid pairs)."""
+    valid = _coerce_pairs(pairs)
+    if not valid:
+        return 0.0
+    return sum((p - o) ** 2 for p, o in valid) / len(valid)
+
+
+def reliability_bins(pairs, n_bins: int | None = None) -> list[ReliabilityBin]:
+    """Equal-width [0,1] reliability bins (the reliability-diagram data). Empty bins are omitted."""
+    if n_bins is None:
+        n_bins = _int_env(_ENV_BINS, _DEFAULT_BINS)
+    n_bins = max(1, n_bins)
+    valid = _coerce_pairs(pairs)
+    buckets: list[list[tuple]] = [[] for _ in range(n_bins)]
+    for p, o in valid:
+        # p == 1.0 lands in the LAST bin (upper-inclusive at the top edge).
+        index = min(n_bins - 1, int(p * n_bins))
+        buckets[index].append((p, o))
+    out: list[ReliabilityBin] = []
+    for i, bucket in enumerate(buckets):
+        if not bucket:
+            continue
+        count = len(bucket)
+        mean_predicted = sum(p for p, _ in bucket) / count
+        mean_outcome = sum(o for _, o in bucket) / count
+        out.append(ReliabilityBin(
+            lower=i / n_bins,
+            upper=(i + 1) / n_bins,
+            count=count,
+            mean_predicted=mean_predicted,
+            mean_outcome=mean_outcome,
+            gap=abs(mean_predicted - mean_outcome),
+        ))
+    return out
+
+
+def calibration_report(pairs, n_bins: int | None = None) -> CalibrationReport:
+    """Full calibration report: n, Brier, ECE (population-weighted mean gap), MCE (max gap), and bins."""
+    valid = _coerce_pairs(pairs)
+    n = len(valid)
+    bins = reliability_bins(valid, n_bins)
+    ece = sum((b.count / n) * b.gap for b in bins) if n else 0.0
+    mce = max((b.gap for b in bins), default=0.0)
+    return CalibrationReport(
+        n=n,
+        brier_score=brier_score(valid),
+        ece=ece,
+        mce=mce,
+        bins=bins,
+    )

+def test_brier_perfect_and_worst():
+    assert brier_score([(0.0, 0), (1.0, 1)]) == 0.0
+    assert brier_score([(0.0, 1), (1.0, 0)]) == 1.0          # worst: ((0-1)^2 + (1-0)^2)/2
+    assert abs(brier_score([(0.7, 1)]) - 0.09) < 1e-9        # (0.7-1)^2
+
+
+def test_perfect_calibration_zero_ece():
+    rep = calibration_report([(0.0, 0), (1.0, 1)])
+    assert rep.n == 2 and rep.brier_score == 0.0 and rep.ece == 0.0 and rep.mce == 0.0
+
+
+def test_overconfident_bin_has_gap():
+    rep = calibration_report([(0.9, 0), (0.9, 0)])           # predicted 0.9, never happened
+    assert abs(rep.brier_score - 0.81) < 1e-9

+    assert abs(rep.bins[0].mean_predicted - 0.7) < 1e-9
+    assert abs(rep.bins[0].mean_outcome - 0.7) < 1e-9

That is still a small lookup table: domain → rubric. The operator constraint is explicit — **replace fixed domain rubrics** with a **generic, domain-agnostic credibility skill the LLM applies by reasoning per question**. Research stream #1's central finding is that the frontier (OpenAI, Gemini, Claude) has already moved this way, and that Perplexity's baked-in EEAT/domain-authority scoring is the closest-to-forbidden counter-example (gameable, popularity-biased, ~60% Google-overlap). LAYER 6 (composition + UI): neutral multi-perspective + transparent evidence-weight artifact, replacing the forewarning/fringe framing\n\n### 2.1 What this REPLACES\n\nThe 2026-06-07 plan's **Phase 7 (L6)** composes contested claims as "consensus at high weight + minority attributed at low weight **with explicit forewarning**, medical-misinfo **low-weight/fringe**, ABSTAIN over fabricated balance." The operator directive is to **replace the forewarning/fringe framing** with a **neutral multi-perspective presentation where competing world-views are shown as legitimate, each carrying its real (possibly asymmetric) evidence weight, and the user judges** — no bias, no stamps, no forewarning prose.\n\n### 2.2 The hard tension, named and resolved (do not skip this)\n\nThere is a real conflict between two binding constraints, and both sides have a lethal failure mode:\n\n- The operator wants **no fringe/forewarning stamp, all sides legitimate, no bias.**\n- The **clinical-safety invariant #5** (binding, preserved) plus Dixon-Clarke 2013 (PubMed 23193194, measured harm: falsely-balanced vaccine reporting *reduced* perceived safety and vaccination intent), Wikipedia FRINGE, and §-1.1 all say **equal treatment of unequal evidence is itself bias** and is "lethal in clinical." I am not authorized to drop invariant #5.\n\n**The reconciliation — handed to us nearly verbatim by all three streams: replace the LABEL with the BASIS.** Wikipedia NPOV's "neutral *editing*, not neutral *content*"; GRADE's "show the basis, let the user infer"; the no-warning rule from stream #2 is satisfiable **if and only if the criteria behind any weight are displayed**. So:\n\n- The **system** stamps nothing — no "fringe," no "debunked," no forewarning sentence. That is the neutrality.\n- The **evidence weight** carries the (often asymmetric) truth, rendered as a visible **per-perspective evidence-basis panel** the user reads and judges. The asymmetry — *14 RCTs + 3 systematic reviews [High]* vs *2 case reports + mechanistic argument [Very Low]* — becomes **visible in the artifact** instead of **asserted in a stamp**. The user infers; the system does not editorialize.\n- "Legitimate" means **shown with its real (possibly low) weight and one-click drill-down to span + verdict** — NOT "shown as equally supported." A genuine world-view dispute with no empirical consensus (values, forecasting) shows steelmanned sides at honest parity; an empirical dispute with consensus shows the real weight distribution.\n- The medical-misinfo case is handled **without the banned word**: its panel is simply **near-empty of independent high-tier basis**, and where there is no independent consensus the claim **ABSTAINs** ("contested; no independent consensus" — reusing existing refuse/unresolved behavior). This preserves invariant #5's *substance* (not normalized as an equal side) while dropping the stamp.\n\nThis is neutral AND anti-false-balance AND §-1.1-safe simultaneously — the operator's "let the user judge" satisfied by *showing the basis*, not by *equalizing the boxes*.\n\n### 2.3 The exact composition mechanism\n\nPer question, the composition path first **diagnoses the contestation type** (this IS the adaptive guardrail, copied from Anthropic's even-handedness "lack-of-consensus" discriminator + OpenAI's seek-the-truth-together):\n\n- **empirical-with-consensus** (vaccines, climate) → due-weight: each side shown, weight proportional to verified-evidence quality; the asymmetry is the message, carried by the basis panel.\n- **genuine-uncertainty** (open scientific questions) → separate *supported* from *open*; sides at honest parity.\n- **values dispute** (politics, ethics) → **steelman each legitimate view** (Ideological Turing Test, neutral terminology, no verdict); no evidence-weight asymmetry asserted because there is no due weight to defer to.\n- **forecasting** (AI-and-jobs) → scenario spread, projective not factual.\n\nEach perspective is gathered through its **own retrieval lens** (MODS-style per-perspective retrieval + citation, arXiv 2502.00322 — raw LLMs silently drop minority views; per-perspective retrieval prevents the drop), then composed via the **existing 4-role / voter-arbiter agreement** (cross-lineage, open-weight voters — the bridging/balanced-panel analog) — **not a new adjudication path**. Reranking-for-balance (arXiv 2506.15925) selects the most balanced faithful candidate, because direct just-prompt-balanced generation empirically skews. **The weight attached to each perspective is evidence QUALITY × independent-origin count, never document count** — else we rebuild Ground News's source-count bias inside the summarizer.\n\n### 2.4 The friendly UI artifact — design\n\nBuilt on the **components that already exist** (#744 citation chip + hover source card, #745 per-claim verdict chip, #747 Proof Replay split-view, #757 report page, #758 claim-anchored compare view) — not designed from scratch.\n\n**The artifact: a claim-anchored "Perspectives" panel** that opens from any contested sentence (reusing the compare-view host), containing per legitimate perspective:\n\n1. **A neutral position header** — "Position held by [who], with this evidence behind it" — no "fringe"/"warning"/"debunked". A steelmanned one-line statement of the view in neutral terms.\n2. **An evidence-basis sub-panel (GRADE-style, the load-bearing element):** the study types / source tiers / independent-origin count behind that perspective, rendered as **frequencies, not a percentage or a lone dot** — "*8 of n: RCTs + systematic reviews [High certainty]*" vs "*2 of n: case reports + mechanistic argument [Very Low]*". The certainty label is acceptable **only because its downgrade criteria are shown** (study design / consistency / directness / independence). This makes the asymmetry self-evident without a stamp.\n3. **Scite-style polarity badges** (supporting / contrasting / contextualizing) on each cited source — **neutral hue + icon + text label, never color alone** (colorblind safety; red=contrasting reads as red=wrong and biases toward consensus — explicitly avoided).\n4. **Drill-down:** every badge, every basis count, every panel is **one click from the raw `[#ev]` span and its VERIFIED/PARTIAL verdict** — the Scite "self-correcting" property: a mis-classified badge is exposed by the citing sentence sitting right behind it, and a meter without drill-down would violate the §-1.1 metadata-audit ban.\n\n**Panel sizing defeats false balance structurally, not with a caveat:** panels are **sized/annotated by real verified-evidence weight**, not equal-width — because equal real estate reads as equal credibility (the false-balance trap), and a caveat is empirically insufficient (Effectiviology / false-balance literature). The asymmetry is *in the layout and the basis panel*, not asserted in prose.\n\n**Three named anti-patterns explicitly rejected** (all from the streams, all as negative templates):\n- **Ground News source-count / coverage bar** — measures who-is-saying-this (volume), not evidence weight; §-1.1 bans source counts as quality signals. If a composition strip is shown at all, its axis is **evidence tier / source type**, never ideology or headcount, and it is kept visually separate from the weight panel so "many sources" never reads as "true."\n- **Bare confidence/credibility score (% or traffic-light dot)** — the best-documented HARM pattern: overconfident displays (80% shown / 70% accurate) pushed AI-advice adoption 57%→69.6%, 64% of users mis-judged calibration, and in a 2025 breast-cancer CDSS high confidence *reduced* diagnostic accuracy via overreliance. POLARIS's honest replacement is the per-sentence VERIFIED/PARTIAL verdict against a real span — never a glanceable dot.\n- **Symmetric 50/50 equal-box presentation** — manufactures false balance; the Dixon-Clarke measured harm.\n\n### 2.5 The honest gap + exactly what POLARIS builds for L6\n\n**HAVE:** the disclosure substrate (per-sentence `[#ev]` tokens, `strict_verify` verdict ladder, `SentenceVerification` additive fields), the existing 4-role/voter-arbiter agreement, independence-collapse for origin-count, and the full UI component kit (citation chip, verdict chip, Proof Replay, compare view).\n**MISSING:** the contestation-type classifier, per-perspective retrieval + composition that defeats false balance by weight, and the Perspectives artifact rendering basis panels.\n**BUILD (revision to Phase 7):**\n1. A **contestation-type classifier** (empirical-consensus / genuine-uncertainty / values / forecasting) priming the composition rule — replaces the fixed forewarning+fringe rule.\n2. **Per-perspective retrieval + balance-rerank composition** over the post-collapse origins, weighting each perspective by verified-evidence quality × independent-origin count (not count).\n3. LAYER 6 (composition + UI): neutral multi-perspective + transparent evidence-weight artifact, replacing the forewarning/fringe framing\n\n### 2.1 What this REPLACES\n\nThe 2026-06-07 plan's **Phase 7 (L6)** composes contested claims as "consensus at high weight + minority attributed at low weight **with explicit forewarning**, medical-misinfo **low-weight/fringe**, ABSTAIN over fabricated balance." The operator directive is to **replace the forewarning/fringe framing** with a **neutral multi-perspective presentation where competing world-views are shown as legitimate, each carrying its real (possibly asymmetric) evidence weight, and the user judges** — no bias, no stamps, no forewarning prose.\n\n### 2.2 The hard tension, named and resolved (do not skip this)\n\nThere is a real conflict between two binding constraints, and both sides have a lethal failure mode:\n\n- The operator wants **no fringe/forewarning stamp, all sides legitimate, no bias.**\n- The **clinical-safety invariant #5** (binding, preserved) plus Dixon-Clarke 2013 (PubMed 23193194, measured harm: falsely-balanced vaccine reporting *reduced* perceived safety and vaccination intent), Wikipedia FRINGE, and §-1.1 all say **equal treatment of unequal evidence is itself bias** and is "lethal in clinical." I am not authorized to drop invariant #5.\n\n**The reconciliation — handed to us nearly verbatim by all three streams: replace the LABEL with the BASIS.** Wikipedia NPOV's "neutral *editing*, not neutral *content*"; GRADE's "show the basis, let the user infer"; the no-warning rule from stream #2 is satisfiable **if and only if the criteria behind any weight are displayed**. So:\n\n- The **system** stamps nothing — no "fringe," no "debunked," no forewarning sentence. That is the neutrality.\n- The **evidence weight** carries the (often asymmetric) truth, rendered as a visible **per-perspective evidence-basis panel** the user reads and judges. The asymmetry — *14 RCTs + 3 systematic reviews [High]* vs *2 case reports + mechanistic argument [Very Low]* — becomes **visible in the artifact** instead of **asserted in a stamp**. The user infers; the system does not editorialize.\n- "Legitimate" means **shown with its real (possibly low) weight and one-click drill-down to span + verdict** — NOT "shown as equally supported." A genuine world-view dispute with no empirical consensus (values, forecasting) shows steelmanned sides at honest parity; an empirical dispute with consensus shows the real weight distribution.\n- The medical-misinfo case is handled **without the banned word**: its panel is simply **near-empty of independent high-tier basis**, and where there is no independent consensus the claim **ABSTAINs** ("contested; no independent consensus" — reusing existing refuse/unresolved behavior). This preserves invariant #5's *substance* (not normalized as an equal side) while dropping the stamp.\n\nThis is neutral AND anti-false-balance AND §-1.1-safe simultaneously — the operator's "let the user judge" satisfied by *showing the basis*, not by *equalizing the boxes*.\n\n### 2.3 The exact composition mechanism\n\nPer question, the composition path first **diagnoses the contestation type** (this IS the adaptive guardrail, copied from Anthropic's even-handedness "lack-of-consensus" discriminator + OpenAI's seek-the-truth-together):\n\n- **empirical-with-consensus** (vaccines, climate) → due-weight: each side shown, weight proportional to verified-evidence quality; the asymmetry is the message, carried by the basis panel.\n- **genuine-uncertainty** (open scientific questions) → separate *supported* from *open*; sides at honest parity.\n- **values dispute** (politics, ethics) → **steelman each legitimate view** (Ideological Turing Test, neutral terminology, no verdict); no evidence-weight asymmetry asserted because there is no due weight to defer to.\n- **forecasting** (AI-and-jobs) → scenario spread, projective not factual.\n\nEach perspective is gathered through its **own retrieval lens** (MODS-style per-perspective retrieval + citation, arXiv 2502.00322 — raw LLMs silently drop minority views; per-perspective retrieval prevents the drop), then composed via the **existing 4-role / voter-arbiter agreement** (cross-lineage, open-weight voters — the bridging/balanced-panel analog) — **not a new adjudication path**.
