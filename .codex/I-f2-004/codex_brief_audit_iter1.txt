## Risks for Codex Red-Team

1. **Test-only route in `app/`.** `web/app/__test__/disambiguation_modal_harness/page.tsx` will be served by the dev server. Production routing concern: `__test__` is conventional Next.js Pages-Router-era keyword for "ignored", but app-router does NOT special-case it. Mitigation: README in `__test__/` directing CI to either remove the directory or set robots/no-index headers in production builds. For dev-only Playwright runs, the harness is fine. If Codex flags this as a P0/P1 risk, brief author commits to using a query-param-driven render branch in an existing demo route OR Playwright Component Testing setup as a follow-up Issue.

2. **No `web/lib/api.ts` import.** The component takes a local `DisambiguationCluster` type to keep this Issue narrow per breakdown. I-f2-005 will refactor to import from `web/lib/api.ts` when it adds `runDisambiguation()`.

3. **Visual-snapshot determinism.** Playwright `toHaveScreenshot()` is sensitive to font rendering + animation timing. Brief recommends `prefers-reduced-motion: reduce` injected via CSS in the harness, AND `await page.waitForFunction(() => !document.querySelector('[data-starting-style]'))` to ensure the modal has finished its starting transition before snapshot. If still flaky, `maxDiffPixels: 100` tolerance (per existing `web/playwright.config.ts` patterns).

4. **DialogPrimitive close-on-outside-click.** Default behavior closes modal when user clicks backdrop. Per existing `ambiguity_modal.tsx:28-30` pattern, `onOpenChange((next_open) => { if (!next_open) onCancel(); })` — same pattern adopted.

5. **Sample-snippet truncation.** 80-char truncation prevents long snippets from breaking layout. Constant `MAX_SAMPLE_LEN = 80` (CLAUDE.md §4.1 — module constant).

6. **`onSelectCluster` invocation.** Brief intent: clicking a card invokes `onSelectCluster(cluster_id)` AND the parent (I-f2-005's intake page) is responsible for closing the modal. Component does NOT close itself on click — single-responsibility (CLAUDE.md §4.2). If Codex argues the modal should close itself for UX, brief author defers since I-f2-005 wires the parent state machine.

7. **`Cancel` button.** Closes the modal via `DialogPrimitive.Close` + invokes `onCancel`. Mirrors `ambiguity_modal.tsx:94-100`.

8. **Empty-clusters defensive render.** Component should never receive empty clusters per I-f2-003 contract (`is_ambiguous=true` only when `num_clusters >= 2`). Defensive: render an empty body, not a runtime error.

9. **Accessibility — focus management.** `<button>` cards are natively focusable. Backdrop dismissal via Esc is provided by DialogPrimitive default. No additional ARIA needed beyond `aria-label` on each card.

10. **Tailwind class drift.** Reuses the exact class strings from `ambiguity_modal.tsx` for backdrop + popup so visual consistency is automatic.

11. **No new dep added to `web/package.json`.** All primitives + utilities already imported elsewhere in the intake module.

12. **Snake_case file naming vs. breakdown.** Breakdown wrote `DisambiguationModal.tsx`; project convention (per `ambiguity_modal.tsx`) is `disambiguation_modal.tsx`. Brief follows project convention; if Codex argues breakdown is binding, brief author renames in iter 2.

type IntakeState =
  | { kind: "idle" }
  | { kind: "loading" }
  | { kind: "ok"; decision: IntakeScopeDecision }
  | { kind: "error"; message: string };

const SAMPLE_QUESTIONS = [
  "Does aspirin reduce headaches in adults?",
  "What are the safety risks of metformin in older adults?",
  "Is physical therapy effective for chronic back pain in adults?",
];

export function IntakeForm() {
  const [question, setQuestion] = useState("");
  const [state, setState] = useState<IntakeState>({ kind: "idle" });
  const [modalOpen, setModalOpen] = useState(false);

  async function submit(e: React.FormEvent<HTMLFormElement>) {
    e.preventDefault();
    const trimmed = question.trim();
    if (trimmed.length < 3) {
      setState({
        kind: "error",
        message: "Please enter a question with at least 3 characters.",
      });
      return;
    }

    setState({ kind: "loading" });
    try {
      const result = await runIntake(trimmed);
      setState({ kind: "ok", decision: result.decision });
      if (result.decision.status === "ambiguous_needs_clarification") {
        setModalOpen(true);
      }
    } catch (err) {
      if (err instanceof IntakeBadRequestError) {
        setState({ kind: "error", message: err.message });
        return;
      }
      const message = err instanceof Error ? err.message : "Unknown error.";
      setState({ kind: "error", message });
    }
  }

      <Card>
        <CardHeader>
          <CardTitle className="text-xl">

            </label>
            <Input
              id="intake-question"
              data-testid="intake-question-input"
              placeholder="e.g. Does aspirin reduce headaches in adults?"
              value={question}
              onChange={(e) => setQuestion(e.target.value)}

            </p>

                variant="default"

import AxeBuilder from "@axe-core/playwright";

/**
 * Phase 2C.5 — WCAG 2.2-AA accessibility audit.
 *
 * For each user-facing route we run axe-core with the WCAG 2A + 2AA + 2.1AA
 * + 2.2AA + best-practice rule sets and assert zero violations. Any new
 * violation should fail CI rather than silently regress accessibility.
 *
 * Failure mode is intentionally LOUD — if axe surfaces a violation we
 * print the rule id, impact, and the affected node selectors so the fix
 * is one click away.
 */

const WCAG_TAGS = [
  "wcag2a",
  "wcag2aa",
  "wcag21a",
  "wcag21aa",
  "wcag22aa",
  "best-practice",
];

async function expectNoA11yViolations(page: import("@playwright/test").Page) {
  const results = await new AxeBuilder({ page }).withTags(WCAG_TAGS).analyze();

  if (results.violations.length > 0) {
    const summary = results.violations
      .map(
        (v) =>
          `\n  - [${v.impact}] ${v.id}: ${v.description}\n    nodes: ${v.nodes
            .map((n) => n.target.join(" ") + (n.failureSummary ? ` — ${n.failureSummary.replace(/\n/g, " ")}` : ""))
            .join("\n           ")}`,
      )
      .join("");
    throw new Error(
      `axe-core found ${results.violations.length} WCAG-AA violation(s):${summary}`,
    );
  }
}
